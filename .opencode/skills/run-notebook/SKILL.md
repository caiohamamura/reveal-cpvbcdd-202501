## What I do

- Execute `.ipynb` Jupyter notebooks via `jupyter nbconvert --execute`
- Verify execution success and report errors
- Extract images from notebook outputs (base64 PNG in JSON) to standalone PNG files
- Support custom execution parameters (e.g., change number of clusters, color palettes)

## When to use me

Use this skill when the user needs to run a Jupyter notebook, verify its outputs, or extract images generated by a notebook for use in slides or documentation.

## How I work

### Phase 1: Execute the notebook

```bash
jupyter nbconvert --execute --to notebook --output output.ipynb \
  --ExecutePreprocessor.timeout=300 \
  notebook_path.ipynb
```

The script handles:
- Timeout configuration (default 300s, adjustable)
- Error capture from cell outputs
- Report generation of success/failure per cell

### Phase 2: Extract images

```bash
python3 .opencode/skills/run-notebook/extract_notebook_images.py \
  notebook_path.ipynb \
  --output-dir images/
```

The extraction script:
1. Reads the `.ipynb` JSON
2. Finds all outputs with `image/png` mime type
3. Decodes base64 data and writes `.png` files
4. Names files sequentially (`cell_{N}_output_{M}.png`) or by a provided mapping

### Phase 3: Override parameters (optional)

The script supports environment variables or inline code injection to change key parameters before execution:
- `N_CLUSTERS` â€” number of clusters
- `PALETTE_STYLE` â€” seaborn/matplotlib style name
- `FIG_SIZE` â€” figure size tuple

Usage:
```bash
N_CLUSTERS=5 python3 .opencode/skills/run-notebook/run_notebook.py notebook.ipynb
```

## Gotchas & Tips

### sklearn dataset conventions
- `load_breast_cancer()`: `target=0` = malignant, `target=1` = benign (counterintuitive â€” 0 is the "abnormal" class). Always verify with `data.target_names`.
- When mapping to anomaly detection where `anomalia_real=1` means anomaly, use `1 - y` to flip.

### Replacing synthetic 2D data with real high-dimensional data
- Use PCA (`sklearn.decomposition.PCA(n_components=2)`) to reduce real datasets to 2D for visualization.
- Standardize with `StandardScaler` **before** PCA (PCA is sensitive to scale).
- Store explained variance ratio to inform students: `pca.explained_variance_ratio_.sum()`.

### Before re-executing a notebook
- Clear old outputs and execution counts via Python script (JSON manipulation) to avoid stale image data confusing students.
- Verify required packages are installed (plotly, scikit-learn, numpy, pandas).

### Editing .ipynb JSON with the edit tool
- **WARNING**: The `edit` tool can consume adjacent cells when oldString spans `"source"` through `"outputs"` blocks. The truncated base64 display (`"... (line truncated ...)"`) in `read` output does NOT match the actual file. Always re-read the JSON after editing to verify cell boundaries.
- Each string in a `"source"` array must end with `\n",` (including the last element â€” no trailing comma on the final `]`).
- When in doubt, rewrite the entire notebook via Python script instead of using multiple `edit` calls.

### Plotly in notebooks (replacing matplotlib)
- Use `plotly.express` for simple scatter plots, `plotly.graph_objects` for complex multi-panel layouts.
- Dracula theme: `template='plotly_dark'` + override `paper_bgcolor`, `plot_bgcolor`, `font`, `gridcolor`.
- `make_subplots(rows, cols, vertical_spacing=N)` controls gap between rows â€” prevents title/x-label overlap.
- Plotly outputs are JSON blobs that inflate file size; clear them between runs.
- `go.Box`: requires `fillcolor` for box fill, `line_color` for border, `marker_color` for outliers.
- `LocalOutlierFactor` with `novelty=True` enables separate `fit()`/`predict()` for training on normal data only.
- `cohen_kappa_score` from sklearn: Kappa=0 means chance-level agreement (useful to expose useless classifiers).

### Anomaly detection pedagogy patterns
- Fit `StandardScaler` and `PCA` **only on benign/normal data** for realistic anomaly detection.
- Train `IsolationForest` and `LOF(novelty=True)` on benign-only; predict on all data.
- Compare TP/FP/FN/TN + F1 + Kappa â€” never rely on recall alone (DBSCAN marking all as anomaly gives 100% recall but Kappa=0).
