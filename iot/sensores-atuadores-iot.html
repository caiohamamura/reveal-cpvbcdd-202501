Bloco 1: Revisão Rápida

Componentes que vocês já conhecem

LDR — Sensor de Luz

  • Resistor que varia com luz → analogRead()
  • Resistor pull-down de 10kΩ forma divisor de tensão
  • Módulo pronto: saída analógica + digital (limiar ajustável)
  • Usos: sensor crepuscular, alarme de claridade
// LDR - leitura analógica int luz = analogRead(A0); // Quanto mais luz, maior o valor if (luz < 200) { Serial.println("Escuro!"); }

DHT22 — Temperatura + Umidade

  • Temperatura + umidade digital
  • Protocolo proprietário (1-wire like)
  • Precisão: ±0.5°C, ±2% UR
  • Faixa: −40°C a +80°C
  • Alta confiabilidade para projetos IoT
// DHT22 com biblioteca DHT #include <DHT.h> DHT dht(DHTPIN, DHT22); float temp = dht.readTemperature(); float umid = dht.readHumidity(); Serial.print("Temp: "); Serial.print(temp); Serial.println(" °C");

LCD I2C + Relé

  • LCD 16x2 I2C: endereço 0x27 ou 0x3F
  • Relé: switch eletromecânico para cargas AC/DC
  • Isolação galvânica entre MCU e carga
  • Relé suporta até 10A/250VAC
// LCD I2C #include <LiquidCrystal_I2C.h> LiquidCrystal_I2C lcd(0x27, 16, 2); lcd.init(); lcd.backlight(); lcd.setCursor(0, 0); lcd.print("Temp: 25.5C"); // Relé digitalWrite(PIN_RELE, HIGH); // liga delay(5000); digitalWrite(PIN_RELE, LOW); // desliga

Motor DC + MOSFET (PWM)

  • MOSFET (IRLZ44N): gate recebe PWM do MCU
  • analogWrite() controla velocidade (0-255)
  • Díodo flyback protege contra EMF reversa
  • MOSFET canal N: fonte → GND, dreno → carga
// Controle de velocidade via PWM int velocidade = 180; // 0-255 analogWrite(PIN_MOTOR, velocidade); // Aceleração gradual for (int i = 0; i <= 255; i++) { analogWrite(PIN_MOTOR, i); delay(10); }

Evoluindo do familiar

TCRT5000 — Sensor Infravermelho Reflexivo

  • Emissor IR + fototransistor no mesmo encapsulamento
  • Deteta superfície clara/escura por reflexão
  • Saída digital (0/1) ou analógica (distância)
  • Usos: seguidor de linha, detector de obstáculo proximal
// TCRT5000 - leitura digital int linha = digitalRead(PIN_TCRT); // LOW = escuro (detetou linha preta) // HIGH = claro (superfície branca) // Leitura analógica int reflexao = analogRead(A0); // Quanto maior, mais reflexão

DS18B20 — Temperatura OneWire

  • Protocolo OneWire (1 fio de dados)
  • À prova d'água (sonda metálica)
  • Precisão: ±0.5°C (−55 a +125°C)
  • Múltiplos sensores no mesmo barramento (ROM ID único)
// DS18B20 com DallasTemperature #include <OneWire.h> #include <DallasTemperature.h> OneWire oneWire(PIN_ONEWIRE); DallasTemperature sensors(&oneWire); sensors.requestTemperatures(); float tempC = sensors.getTempCByIndex(0); Serial.print(tempC); Serial.println(" °C");

TMP36 — Temperatura Analógica

  • Saída analógica linear: 10 mV/°C
  • Faixa: −40°C a +125°C
  • Offset 500 mV = 0°C
  • Barato e simples, porém menos preciso que DS18B20/DHT22
// TMP36 - conversão analógica int raw = analogRead(A0); float voltage = raw * (3.3 / 1024.0); float tempC = (voltage - 0.5) * 100; Serial.print("Temp: "); Serial.print(tempC); Serial.println(" °C");

Bloco 3: Sensores de Distância

Ultrasom, laser e radar

HC-SR04 — Ultrassônico

  • Emite pulso ultrassônico (40 kHz)
  • Mede tempo do eco → distância
  • Faixa: 2 cm a 4 m
  • Precisão: ~3 mm
  • 2 pinos: TRIG (saída) + ECHO (entrada)
// HC-SR04 com NewPing #include <NewPing.h> NewPing sonar(TRIG, ECHO, 400); void loop() { int dist = sonar.ping_cm(); Serial.print("Distância: "); Serial.print(dist); Serial.println(" cm"); delay(500); }

Bloco 4: Sensores Ambientais

Gás, solo, vibração, chuva, aceleração

MQ-2 — Gás Inflamável / Fumaça

graph LR
  GAS[Gás GLP] --> MQ2[MQ-2]
  MQ2 -->|"A0"| MCU
  MCU -->|"ppm > limiar?"| ALERT["🚨 Alarme"]
  style MQ2 fill:#ff5555,color:#f8f8f2
  • Deteta GLP, metano, álcool, fumaça
  • Saída analógica (ppm proporcional)
  • Potenciômetro ajusta sensibilidade
  • Aquecimento: precisa de ~20 s para estabilizar
// MQ-2 - leitura analógica int gas = analogRead(A0); if (gas > LIMIAR) { digitalWrite(PIN_BUZZER, HIGH); digitalWrite(PIN_LED, HIGH); Serial.println("ALERTA: Gás!"); } else { digitalWrite(PIN_BUZZER, LOW); digitalWrite(PIN_LED, LOW); }

Capacitive Soil Moisture

  • Mede umidade do solo por capacitância
  • Não oxida (diferente do resistivo)
  • Saída analógica: seco ~530, molhado ~260
  • Ideal: irrigação automática
// Capacitive Soil Moisture int raw = analogRead(A0); int umidade = map(raw, 530, 260, 0, 100); umidade = constrain(umidade, 0, 100); Serial.print("Umidade: "); Serial.print(umidade); Serial.println("%");

MPU-6050 — Acelerômetro + Giroscópio

  • 6 eixos: 3 accel + 3 gyro
  • I2C (endereço 0x68)
  • Aceleração: ±2/4/8/16g
  • Giroscópio: ±250/500/1000/2000°/s
  • Usos: drone, robô balanceiro, IMU
// MPU-6050 com Adafruit_MPU6050 #include <Adafruit_MPU6050.h> Adafruit_MPU6050 mpu; void loop() { sensors_event_t a, g, temp; mpu.getEvent(&a, &g, &temp); Serial.print("Accel X: "); Serial.print(a.acceleration.x); Serial.println(" m/s²"); delay(500); }

Sensor Hall — Campo Magnético

  • US1881: digital (liga/desliga com ímã)
  • SS49E: analógico (intensidade do campo)
  • Usos: velocímetro (ímã no eixo), contador de voltas
// US1881 - contador de voltas int campo = digitalRead(PIN_HALL); static int voltas = 0; static unsigned long t = 0; if (campo == LOW && millis() - t > 100) { voltas++; t = millis(); Serial.print("Voltas: "); Serial.println(voltas); }

PIR HC-SR501 — Presença (IV)

  • Deteta calor infravermelho em movimento
  • Faixa: ~7 m, ângulo 120°
  • 2 potenciômetros: sensibilidade e tempo
  • Usos: alarme, iluminação automática
// PIR HC-SR501 int presenca = digitalRead(PIN_PIR); if (presenca == HIGH) { digitalWrite(PIN_LED, HIGH); Serial.println("Presença!"); delay(10000); } else { digitalWrite(PIN_LED, LOW); }

Bloco 5: Sensores de Corrente e Tensão

Monitoramento energético

INA219 — Corrente e Tensão (I2C)

  • Mede corrente via shunt integrado
  • Mede tensão do barramento
  • I2C (0x40 ou 0x41)
  • Faixa: até 26V, ±3.2A
  • Ideal: monitorar consumo de projetos IoT
// INA219 com Adafruit_INA219 #include <Adafruit_INA219.h> Adafruit_INA219 ina219; void loop() { float busV = ina219.getBusVoltage_V(); float current = ina219.getCurrent_mA(); float power = busV * (current / 1000.0); Serial.print("Tensão: "); Serial.print(busV); Serial.println(" V"); delay(1000); }

Divisor de Tensão — Medição de Tensão

graph LR
  VIN["Bateria 12V"] --> R1[100kΩ]
  R1 --> VOUT["Vout (ADC)"]
  VOUT --> R2[33kΩ]
  R2 --> GND
  style R1 fill:#ff5555,color:#f8f8f2
  style R2 fill:#8be9fd,color:#282a36
  • 2 resistores em série: Vout = Vin × R2/(R1+R2)
  • Permite ler tensões acima de VCC
  • Ex: bateria 12V → ADC 3.3V (fator 4.03)
  • Resistores altos (> 10kΩ) reduzem consumo
// Divisor R1=100k, R2=33k float fator = (100.0 + 33.0) / 33.0; int raw = analogRead(A0); float vout = raw * (3.3 / 1024.0); float vin = vout * fator; Serial.print("Bateria: "); Serial.print(vin); Serial.println(" V");

Bloco 6: Atuadores — Motores e Movimento

Controlando o mundo físico

SG90 — Servo Motor

  • Rotação precisa: 0° a 180°
  • Controle via PWM (biblioteca Servo)
  • Torque: ~1.8 kg·cm (4.8V)
  • Usos: braço robótico, fechadura, câmera pan/tilt
#include <Servo.h> Servo myservo; void setup() { myservo.attach(PIN_SERVO); } void loop() { myservo.write(0); // 0 graus delay(1000); myservo.write(90); // 90 graus delay(1000); myservo.write(180); // 180 graus delay(1000); }

28BYJ-48 + ULN2003 — Motor de Passo

  • Motor de passo 5V, 4 fases
  • Driver ULN2003 (transistores Darlington)
  • 2048 passos/volta (half-step)
  • Usos: dispensador, cortina automática, foco de câmera
#include <Stepper.h> #define STEPS 2048 Stepper stepper(STEPS, 8, 10, 9, 11); void setup() { stepper.setSpeed(10); // RPM } void loop() { stepper.step(512); // 1/4 volta delay(1000); stepper.step(-512); // volta delay(1000); }

WS2812B — LED Endereçável (Neopixel)

  • LED RGB com controlador integrado
  • 1 pino de dados controla N LEDs (daisy-chain)
  • Cada LED: 16M cores, endereço individual
  • Usos: indicação visual, fita decorativa, matriz de LEDs
#include <Adafruit_NeoPixel.h> #define PIN 6 #define NUM 16 Adafruit_NeoPixel strip(NUM, PIN, NEO_GRB); void setup() { strip.begin(); strip.setBrightness(50); } void loop() { strip.setPixelColor(0, strip.Color(255, 0, 0)); // vermelho strip.setPixelColor(1, strip.Color(0, 255, 0)); // verde strip.show(); delay(500); }

Solenóide + Válvula Solenóide

  • Solenóide: eletroímã linear (puxa/empurra)
  • Válvula solenóide: abre/fecha fluxo de líquido
  • Controlados via MOSFET ou relé
  • Atenção: consomem muita corrente (~500mA+)
  • Usos: fechadura elétrica, irrigação
// Solenóide via MOSFET digitalWrite(PIN_SOL, HIGH); // ativa delay(3000); digitalWrite(PIN_SOL, LOW); // desativa // Válvula solenóide // HIGH = água passa // LOW = bloqueia

MQTT — Protocolo IoT por Excelência

graph LR

ESP8266 — Wi-Fi Nativo + HTTP

  • Wi-Fi integrado (802.11 b/g/n)
  • HTTP GET/POST para APIs REST
  • Compatível com ThingsBoard, Blynk, Firebase
  • Substitui Arduino + shield Ethernet
// ESP8266 HTTP GET #include <ESP8266WiFi.h> #include <ESP8266HTTPClient.h> HTTPClient http; http.begin("http://api.exemplo.com" "/dados?temp=" + String(temp)); int code = http.GET(); if (code > 0) { String payload = http.getString(); } http.end();
