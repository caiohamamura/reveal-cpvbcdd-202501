Bloco 1: Revisão Rápida

Componentes que vocês já conhecem

LDR + DHT22 (Revisão)

  • LDR: resistor que varia com luz → analogRead()
  • DHT22: temperatura + umidade digital
  • Protocolo proprietário (1-wire like)
  • Precisão DHT22: ±0.5°C, ±2% UR
// LDR - leitura analógica int luz = analogRead(A0); // DHT22 - leitura digital float temp = dht.readTemperature(); float umid = dht.readHumidity();

LCD I2C + Relé (Revisão)

  • LCD 16x2 I2C: endereço 0x27 ou 0x3F
  • Relé: switch eletromecânico para cargas AC/DC
  • Relé = isolação galvânica entre MCU e carga
// LCD I2C lcd.init(); lcd.backlight(); lcd.setCursor(0, 0); lcd.print("Temp: "); lcd.print(temp); // Relé digitalWrite(PIN_RELE, HIGH); // liga digitalWrite(PIN_RELE, LOW); // desliga

Motor DC + MOSFET (Revisão)

  • MOSFET (IRLZ44N): gate recebe PWM do MCU
  • analogWrite() controla velocidade (0-255)
  • Díodo flyback protege contra EMF reversa
// Controle de velocidade via PWM int velocidade = 180; // 0-255 analogWrite(PIN_MOTOR, velocidade); // Parar motor analogWrite(PIN_MOTOR, 0);

Bloco 2: Sensores de Luz e Temperatura

Evoluindo do familiar

TCRT5000 — Sensor Infravermelho Reflexivo

  • Emissor IR + fototransistor
  • Deteta superfície clara/escura
  • Saída digital (0/1) ou analógica
  • Usos: seguidor de linha, detector de obstáculo
// TCRT5000 - leitura digital int linha = digitalRead(PIN_TCRT); // LOW = superfície escura (linha) // HIGH = superfície clara // Leitura analógica (distância) int reflexao = analogRead(A0);

DS18B20 — Temperatura OneWire

  • Protocolo OneWire (1 fio de dados)
  • À prova d'água (sonda metálica)
  • Precisão: ±0.5°C (−55 a +125°C)
  • Múltiplos sensores no mesmo barramento
// DS18B20 com DallasTemperature OneWire oneWire(PIN_ONEWIRE); DallasTemperature sensors(&oneWire); sensors.requestTemperatures(); float tempC = sensors.getTempCByIndex(0);

TMP36 — Temperatura Analógica

  • Saída analógica linear: 10 mV/°C
  • Faixa: −40°C a +125°C
  • Sem protocolo — direto no ADC
  • Barato e simples, porém menos preciso
// TMP36 - conversão analógica int raw = analogRead(A0); float voltage = raw * (5.0 / 1023.0); float tempC = (voltage - 0.5) * 100; // 500mV offset = 0°C

Bloco 3: Sensores de Distância

Ultrasom, laser e radar

HC-SR04 — Ultrassônico

  • Emite pulso ultrassônico (40 kHz)
  • Mede tempo do eco → distância
  • Faixa: 2 cm a 4 m
  • Precisão: ~3 mm
  • 2 pinos: TRIG (saída) + ECHO (entrada)
// HC-SR04 com NewPing #include <NewPing.h> NewPing sonar(TRIG, ECHO, 400); void loop() { int dist = sonar.ping_cm(); Serial.print("Distância: "); Serial.print(dist); Serial.println(" cm"); }

VL53L0X — ToF Laser

  • Time-of-Flight a laser infravermelho
  • I2C (endereço 0x29)
  • Faixa: 30 mm a 1.2 m
  • Muito mais preciso que ultrassônico
  • Ideal: robótica, drone, nível de tanque
// VL53L0X com Adafruit_VL53L0X #include <Adafruit_VL53L0X.h> Adafruit_VL53L0X lox = Adafruit_VL53L0X(); void loop() { VL53L0X_RangingMeasurementData_t measure; lox.rangingTest(&measure, false); if (measure.RangeStatus != 4) { Serial.print("Dist: "); Serial.print(measure.RangeMilliMeter); Serial.println(" mm"); } }

RCWL-0516 — Radar Doppler

  • Radar de micro-ondas 3.2 GHz
  • Deteta movimento através de paredes/vidro
  • Faixa: ~7 m (ajustável)
  • Saída digital (HIGH = movimento)
  • Não depende de calor (diferente do PIR)
// RCWL-0516 - leitura digital int movimento = digitalRead(PIN_RCWL); if (movimento == HIGH) { Serial.println("Movimento detetado!"); }

Bloco 4: Sensores Ambientais

Gás, solo, vibração, chuva, aceleração

MQ-2 — Gás Inflamável / Fumaça

  • Deteta GLP, metano, álcool, fumaça
  • Saída analógica (ppm proporcional)
  • Potenciômetro ajusta sensibilidade
  • Aquecimento: precisa de ~20 s para estabilizar
// MQ-2 - leitura analógica int gas = analogRead(A0); // Conversão aproximada para ppm float ppm = gas * (5.0 / 1023.0); if (ppm > LIMIAR) { Serial.println("ALERTA: Gás detetado!"); }

Capacitive Soil Moisture

  • Mede umidade do solo por capacitância
  • Não oxida (diferente do resistivo)
  • Saída analógica: seco ~530, molhado ~260
  • Tensão: 3.3–5.5V
  • Ideal: irrigação automática
// Capacitive Soil Moisture int raw = analogRead(A0); // Mapear para porcentagem int umidade = map(raw, 530, 260, 0, 100); umidade = constrain(umidade, 0, 100); Serial.print("Umidade: "); Serial.print(umidade); Serial.println("%");

SW-420 — Vibração

  • Switch de mercúrio / mola vibrátil
  • Digital: HIGH quando vibra
  • Potenciômetro ajusta sensibilidade
  • Usos: alarme, deteção de terremoto
// SW-420 - deteção de vibração int vib = digitalRead(PIN_SW420); if (vib == HIGH) { Serial.println("Vibração detetada!"); }

Sensor de Chuva

  • Placa com trilhas resistivas
  • Água fecha circuito → resistência cai
  • Saída analógica (intensidade) e digital (limiar)
  • Proteger conexões da água!
// Sensor de chuva int chuva = analogRead(A0); // Valores altos = seco, baixos = chuva int intensidade = map(chuva, 1023, 200, 0, 100); Serial.print("Chuva: "); Serial.print(intensidade); Serial.println("%");

MPU-6050 — Acelerômetro + Giroscópio

  • 6 eixos: 3 accel + 3 gyro
  • I2C (endereço 0x68)
  • Aceleração: ±2/4/8/16g
  • Giroscópio: ±250/500/1000/2000°/s
  • Usos: drone, robô balanceiro, IMU
// MPU-6050 com Adafruit_MPU6050 #include <Adafruit_MPU6050.h> Adafruit_MPU6050 mpu; void loop() { sensors_event_t a, g, temp; mpu.getEvent(&a, &g, &temp); Serial.print("Accel X: "); Serial.print(a.acceleration.x); Serial.println(" m/s²"); }

Sensor Hall — Campo Magnético

  • US1881: digital (on/off)
  • SS49E: analógico (intensidade)
  • Deteta ímãs / campo magnético
  • Usos: velocímetro (ímã no eixo), contador de voltas
// US1881 - digital int campo = digitalRead(PIN_HALL); // SS49E - analógico int intensidade = analogRead(A0); // Contar voltas com ímã if (campo == LOW) { voltas++; }

PIR HC-SR501 — Presença

  • Deteta calor infravermelho em movimento
  • Faixa: ~7 m, ângulo 120°
  • 2 potenciômetros: sensibilidade e tempo
  • Saída digital (HIGH = presença)
  • Usos: alarme, iluminação automática
// PIR HC-SR501 int presenca = digitalRead(PIN_PIR); if (presenca == HIGH) { digitalWrite(PIN_LED, HIGH); Serial.println("Presença detetada!"); } else { digitalWrite(PIN_LED, LOW); }

Bloco 5: Sensores de Corrente e Tensão

Monitoramento energético

INA219 — Corrente e Tensão (I2C)

  • Mede corrente via shunt integrado
  • Mede tensão do barramento
  • I2C (0x40 ou 0x41)
  • Faixa: até 26V, ±3.2A
  • Precisão: 0.1 mA
  • Ideal: monitorar consumo de projetos IoT
// INA219 com Adafruit_INA219 #include <Adafruit_INA219.h> Adafruit_INA219 ina219; void loop() { float shuntV = ina219.getShuntVoltage_mV(); float busV = ina219.getBusVoltage_V(); float current = ina219.getCurrent_mA(); float power = busV * (current / 1000.0); Serial.print("Potência: "); Serial.print(power); Serial.println(" W"); }

Divisor de Tensão — Medição de Tensão

  • 2 resistores em série: Vout = Vin × R2/(R1+R2)
  • Permite ler tensões acima de VCC
  • Ex: medir bateria 12V com ADC 3.3V
  • Atenção: resistores afetam consumo!
// Divisor de tensão (R1=100k, R2=33k) // Fator: 133k/33k = 4.03 float fator = (100.0 + 33.0) / 33.0; int raw = analogRead(A0); float vout = raw * (3.3 / 1023.0); float vin = vout * fator; Serial.print("Tensão: "); Serial.print(vin); Serial.println(" V");

Bloco 6: Atuadores — Motores e Movimento

Controlando o mundo físico

SG90 — Servo Motor

  • Rotação precisa: 0° a 180°
  • Controle via PWM (servo library)
  • Torque: ~1.8 kg·cm (4.8V)
  • Usos: braço robótico, fechadura, ventoinha direcional
#include <Servo.h> Servo myservo; void setup() { myservo.attach(PIN_SERVO); } void loop() { myservo.write(0); // 0 graus delay(1000); myservo.write(90); // 90 graus delay(1000); myservo.write(180); // 180 graus delay(1000); }

28BYJ-48 + ULN2003 — Motor de Passo

  • Motor de passo 5V, 4 fases
  • Driver ULN2003 (darlington array)
  • 2048 passos/volta (half-step)
  • Rotação precisa em passos discretos
  • Usos: dispensador, cortina automática
#include <Stepper.h> #define STEPS 2048 Stepper stepper(STEPS, 8, 10, 9, 11); void setup() { stepper.setSpeed(10); // RPM } void loop() { stepper.step(512); // 1/4 volta delay(1000); stepper.step(-512); // volta delay(1000); }

WS2812B — LED Endereçável (Neopixel)

  • LED RGB com controlador integrado
  • 1 pino de dados controla N LEDs
  • Cada LED: 16M cores, endereço individual
  • Cascateável (daisy-chain)
  • Usos: indicação visual, fita decorativa
#include <Adafruit_NeoPixel.h> #define PIN 6 #define NUM 16 Adafruit_NeoPixel strip(NUM, PIN, NEO_GRB); void setup() { strip.begin(); strip.setBrightness(50); // 0-255 } void loop() { strip.setPixelColor(0, strip.Color(255, 0, 0)); // LED 0 = vermelho strip.show(); }

Solenóide + Válvula Solenóide

  • Solenóide: eletroímã linear (puxa/empurra)
  • Válvula solenóide: abre/fecha fluxo de líquido
  • Controlados via MOSFET ou relé
  • Atenção: consomem muita corrente (~500mA+)
  • Usos: fechadura elétrica, irrigação
// Solenóide via MOSFET (N-channel) // Gate → pino MCU via resistor 220Ω // Drain → solenóide negativo // Source → GND digitalWrite(PIN_SOLENOIDE, HIGH); // abre delay(3000); digitalWrite(PIN_SOLENOIDE, LOW); // fecha // Válvula solenóide funciona igual // HIGH = água passa, LOW = bloqueia

Bloco 7: Displays Alternativos

Além do LCD 16x2

OLED SSD1306 — Display 128×64 (I2C)

  • Display gráfico monocromático
  • 128×64 pixels, alto contraste
  • I2C (0x3C ou 0x3D)
  • Texto, formas, bitmaps
  • Consome muito pouco (~10 mA)
#include <Adafruit_SSD1306.h> #define W 128 #define H 64 Adafruit_SSD1306 display(W, H, &Wire, -1); void setup() { display.begin(SSD1306_SWITCHCAPVCC, 0x3C); display.clearDisplay(); display.setTextSize(2); display.setTextColor(WHITE); display.setCursor(0, 0); display.println("IoT!"); display.display(); }

TM1637 — Display 4 Dígitos

  • 4 dígitos 7 segmentos + 2 pontos
  • Protocolo próprio (2 fios: CLK + DIO)
  • Ideal para mostrar: relógio, temperatura, contagem
  • Muito barato e simples
#include <TM1637Display.h> TM1637Display display(CLK, DIO); void setup() { display.setBrightness(0x0F); // max } void loop() { int temp = dht.readTemperature(); display.showNumberDec(temp); // Mostra: " 25" se temp=25 delay(2000); }

Protocolos e conectividade

Protocolos: I2C vs SPI vs OneWire vs UART

MQTT — Protocolo IoT por Excelência

  • Publish/Subscribe via broker
  • Leve: header mínimo de 2 bytes
  • Tópicos hierárquicos: casa/sala/temp
  • QoS: 0 (fire&forget), 1 (pelo menos 1), 2 (exatamente 1)
  • Broker popular: Mosquitto
// ESP8266 + PubSubClient #include <PubSubClient.h> WiFiClient espClient; PubSubClient client(espClient); void setup() { client.setServer("broker.hivemq.com", 1883); } void loop() { char msg[8]; dtostrf(temp, 1, 1, msg); client.publish("casa/sala/temp", msg); }

ESP8266 — Wi-Fi Nativo + HTTP

  • Wi-Fi integrado (802.11 b/g/n)
  • HTTP GET/POST para APIs
  • REST API: ThingsBoard, Blynk, Firebase
  • Substitui Arduino + shield Ethernet
// ESP8266 HTTP GET #include <ESP8266WiFi.h> #include <ESP8266HTTPClient.h> HTTPClient http; http.begin("http://api.exemplo.com" "/dados?temp=" + String(temp)); int code = http.GET(); if (code > 0) { String payload = http.getString(); } http.end();
