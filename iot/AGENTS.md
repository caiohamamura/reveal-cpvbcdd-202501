# AGENTS.md — IoT Course Slides

This directory contains Reveal.js slide decks for the IoT course (IFSP Campus Capivari).

## Directory Scope

This workspace is restricted to `iot/`. For shared dependencies (components, CSS, templates, plugins), reference the root:
- `../components/components.js` — Vue components
- `../dist/custom.css` — Dracula theme overrides
- `../slides_template/init.js` — `mountSlideApp()` initializer
- `../slides_template/header1.js` — `<header1>` cover component
- `../plugin/` — Reveal.js plugins
- `../dist/vue.js` — Vue 3 ESM browser build
- `../dist/reveal.js` — Reveal.js 5.2.1

### Aula 13 variant family
17 variants of "Sensores e Atuadores para Projetos IoT", each generated by a different AI model:

```
13-sensores-atuadores-{MODEL}[-noplan].html
```

- `MODEL` = AI model name (qwen, gpt5.3-codex, gemini31prohigh, glm, deepseekv4pro, minimax, etc.)
- `-noplan` suffix = generated without planning/outline step
- No suffix = planned version (outline first, then slides)

## Styling

- **Dracula palette**: bg `#282a36`, fg `#f8f8f2`, cyan `#8be9fd`, green `#50fa7b`, pink `#ff79c6`, red `#ff5555`, yellow `#f1fa8c`, purple `#bd93f9`, comment `#6272a4`, current line `#44475a`
- **Typography**: `<h2>` = 42pt Oswald, `<h3>` = 32pt green with shadow, `<h4>` = 24pt, body/`<li>` = 22pt, all left-aligned
- Use inline `style=""` for one-off styling

## Critical Rules

- **4-space indentation**: ALL content inside `<div class="slides">`
- **Code blocks**: ALWAYS wrap in `<script type="text/plain">` to prevent HTML parser mangling. Use `lang="cpp"` for Arduino/C++, `lang="sql"` for SQL, `lang="python"` for Python
- **Leader-line**: MUST use explicit close tag `</leader-line>` — self-closing `<leader-line />` breaks HTML parser
- **Images**: ALL `<img>` must have `alt` text. Use `width` OR `height`, not both. Center with parent `style="text-align: center"`. Include source attribution (`Fonte: [Site] ([License])`)
- **Language**: All slide content in Brazilian Portuguese. No Chinese characters. English technical terms are fine.
- **Student-facing**: Do not mention `micromamba`, `opencode`, or internal setup in student-facing content
- **No comments in code**: Do not add comments unless explicitly requested

## Fragment Usage

- Use fragments SPARINGLY — only for step-by-step concepts, code walkthroughs, exercise steps
- Use `<ls-u>` (auto-fragment) ONLY for: motivation slides, exercise steps, summaries
- Use plain `<ul>` for: definitions, reference lists, informational content

## Content Flow (per lesson)

1. Cover (`<header1>`)
2. Review (previous lesson recap)
3. Motivation (why topic matters)
4. Concept sections (2-5 slides per concept)
5. Practical exercise
6. Summary

## Super Sections

- Wrap major topics in parent `<section>` for vertical navigation
- Cover and review slides stay OUTSIDE super sections (top-level)
- Close with `<!-- fim {NAME} -->` comment
