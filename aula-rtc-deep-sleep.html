🔋 O Desafio da Bateria

  • ESP8266 ligado consome ~80 mA
  • Pilha AA alcalina: ~2500 mAh
  • Autonomia: 2500 ÷ 80 = ~31 horas
  • E se eu precisasse de 6 meses no meio do mato?
    •

    💤 Deep Sleep = ~20 μA

  • De 80 mA → 0,02 mA
  • 4.000x menos consumo!
  • Autonomia: de 31 horas → anos
  • O segredo: dormir mais, acordar menos
    •

    💤 Como o ESP8266 Dorme

    Modo Consumo Acordar com Uso Típico
    Modem Sleep ~15 mA Timer automático Wi-Fi off entre tx
    Light Sleep ~0,4 mA Timer ou GPIO CPU pausada
    Deep Sleep ~20 μA (0,02 mA) GPIO 16 (RST) Tudo desligado

    💤 Deep Sleep em Detalhe

  • Tudo é desligado (CPU, Wi-Fi, RAM)
  • Só o RTC continua rodando
  • Consome apenas 20 μA (0,02 mA)
  • Acorda como se tivesse sido ligado do zero
  • Roda setup() novamente
    •

    Analogia: É como desligar o celular. O despertador continua, mas tudo mais reinicia.

    ⚡ O Pino Mágico: GPIO 16 → RST

  • Conexão obrigatória: D0 (GPIO 16) → RST
  • É o RTC que "cutuca" o RST para acordar
  • Sem esse fio: o ESP dorme para sempre
    •

    🔢 Calculadora de Autonomia

    Dados:

  • Bateria 18650: 3000 mAh
  • Acordado (Wi-Fi): 80 mA por 5 seg
  • Dormindo: 0,02 mA
  • Ciclo: a cada 30 minutos
    •

    Cálculo:

  • 48 ciclos/dia
  • Acordado/dia: 80 × 4/60h = 5,3 mAh
  • Dormindo/dia: 0,02 × 23,93h = 0,48 mAh
  • Total/dia: ~5,8 mAh
    •

    Autonomia: 3000 ÷ 5,8 ≈ 517 dias! 🤯

    🕐 O Problema: "Que Horas São?"

  • Deep sleep reinicia TUDO
  • Variáveis normais são perdidas
  • Hora, contadores, dados... tudo some!

    Como lembrar que horas são sem Wi-Fi?

    📝 RTC Memory — O "Post-it" do ESP8266

  • 512 bytes que sobrevivem ao deep sleep
  • Declarado com RTC_DATA_ATTR
  • Rápido como RAM (sem desgaste de flash)
  • Não sobrevive a desconexão de energia
    •
    RTC_DATA_ATTR int contador = 0; // Persiste! RTC_DATA_ATTR float ultimaTemperatura = 0; // Persiste!

    🧠 O Que Sobrevive?

    Tipo Deep Sleep? Power-off? Velocidade
    RAM normal ❌ Perde ❌ Perde Rápida
    RTC memory ✅ Mantém ❌ Perde Rápida
    EEPROM ✅ Mantém ✅ Mantém Média
    SPIFFS ✅ Mantém ✅ Mantém Lenta

    ⏰ NTP Inteligente

  • Primeira vez: conecta Wi-Fi → sincroniza NTP → salva epoch
  • Depois: calcula hora via millis() - millisBase + epochBase
  • Economia: não conecta Wi-Fi a cada ciclo!
    •

    5 seg de Wi-Fi = 5 seg × 80mA. Sem Wi-Fi = só 5ms × 20mA!

    ⏰ Código: NTP + RTC Memory

    RTC_DATA_ATTR bool ntpOk = false; RTC_DATA_ATTR time_t epochBase = 0; RTC_DATA_ATTR unsigned long millisBase = 0; void setup() { if (!ntpOk) { WiFi.begin(ssid, senha); // ... sincroniza NTP ... epochBase = time(nullptr); millisBase = millis(); ntpOk = true; WiFi.disconnect(true); } time_t agora = epochBase + (millis() - millisBase) / 1000; Serial.printf("Hora: %s", ctime(&agora)); ESP.deepSleep(300e6); // 5 min }
