    Parte 1: Entrando no MongoDB

    Banco, coleção, documento e BSON

    Objetivo da Aula

    Ao final, você deve conseguir:

    • Carregar uma base simples no MongoDB
    • Consultar documentos com filtros e projeções
    • Atualizar documentos com operadores
    • Criar índice e observar plano de execução
    • Montar uma agregação básica

    BSON e _id

    JSON

    • Formato textual
    • Ótimo para APIs e leitura humana
    • Tipos limitados

    BSON

    • Formato binário do MongoDB
    • Inclui tipos como ObjectId e datas
    • O campo _id identifica cada documento

    A estrutura parece JSON, mas o banco armazena e consulta BSON.

    Parte 2: CRUD e Consultas

    Inserir, buscar, projetar, atualizar e remover documentos

    Inserindo Documentos

    db.produtos.insertOne({ sku: "AC-100", nome: "Mouse Vertical", categoria: "acessorio", preco: 189.9, estoque: 45, specs: { conexao: "Bluetooth", dpi: 3200 }, tags: ["ergonomia", "escritorio"] })

    A coleção não exigiu CREATE TABLE. Isso é produtivo, mas também permite inconsistência se a aplicação não controlar o formato.

    Primeiras Consultas

    db.produtos.find() db.produtos.find({ categoria: "notebook" }) db.produtos.find( { categoria: "notebook" }, { nome: 1, preco: 1, estoque: 1 } )

    O segundo argumento é a projeção: quais campos aparecem no resultado.

    Filtros com Operadores

    db.produtos.find({ preco: { $gte: 1000, $lte: 6000 } }) db.produtos.find({ categoria: { $in: ["notebook", "celular"] } }) db.produtos.find({ $or: [ { estoque: { $lt: 10 } }, { tags: "promocao" } ] })

    Campos Aninhados

    A dot notation consulta campos dentro de objetos:

    db.produtos.find({ "specs.ram_gb": { $gte: 16 } }) db.produtos.find({ "specs.camera_mp": { $gte: 48 } })

    Esse é um dos motivos de MongoDB parecer natural para objetos da aplicação.

    Arrays

    db.produtos.find({ tags: "promocao" }) db.produtos.find({ tags: { $all: ["trabalho", "portatil"] } }) db.produtos.find({ tags: { $size: 2 } })

    Se um campo é array, uma busca por valor simples encontra documentos onde o array contém aquele valor.

    Atualizações

    db.produtos.updateOne( { sku: "NB-001" }, { $set: { preco: 4999.9 } } ) db.produtos.updateOne( { sku: "NB-001" }, { $inc: { estoque: -1 } } ) db.produtos.updateOne( { sku: "NB-001" }, { $push: { tags: "oferta-relampago" } } )

    Cuidado com Updates Amplos

    Filtro vazio altera muitos documentos:

    db.produtos.updateMany( {}, { $set: { revisado: true } } )

    Antes de atualizar ou remover, execute o filtro com find() e confirme o conjunto afetado.

    Remoção

    db.produtos.deleteOne({ sku: "AC-100" }) db.produtos.deleteMany({ categoria: "acessorio", estoque: 0 })

    A regra prática é a mesma do SQL: remova com filtros específicos e verificáveis.

    Parte 3: Índices e Plano de Execução

    Consultas flexíveis também precisam de estrutura

    Consulta Sem Índice

    db.produtos.find({ categoria: "notebook", preco: { $lte: 6000 } }).explain("executionStats")

    Procure por COLLSCAN: o banco examinou a coleção sequencialmente.

    Criando Índice

    db.produtos.createIndex({ categoria: 1, preco: 1 }) db.produtos.find({ categoria: "notebook", preco: { $lte: 6000 } }).explain("executionStats")

    Agora procure por IXSCAN: a busca usou índice.

    Índice em Campo Aninhado

    db.produtos.createIndex({ "specs.ram_gb": 1 }) db.produtos.find({ "specs.ram_gb": { $gte: 16 } })

    MongoDB indexa campos aninhados, mas o índice só ajuda se a consulta combina com o padrão de acesso.

    Índice Não É Decoração

    DecisãoBenefícioCusto
    Índice simplesAcelera filtro comumOcupa espaço e pesa escrita
    Índice compostoAjuda filtros combinadosOrdem dos campos importa
    Índice em campo aninhadoAcelera dot notationDepende da estabilidade do formato
    Muitos índicesMais caminhos de buscaInserções e updates ficam mais caros

    O equivalente documental de filtrar, transformar e agrupar

    Preço Médio por Categoria

    db.produtos.aggregate([ { $match: { estoque: { $gt: 0 } } }, { $group: { _id: "$categoria", total: { $sum: 1 }, precoMedio: { $avg: "$preco" } } }, { $sort: { precoMedio: -1 } } ])

    Projetando o Resultado

    db.produtos.aggregate([ { $match: { categoria: "notebook" } }, { $project: { _id: 0, nome: 1, preco: 1, ram: "$specs.ram_gb", ssd: "$specs.ssd_gb" } }, { $sort: { ram: -1, preco: 1 } } ])

    Agrupando Pedidos

    db.pedidos.aggregate([ { $unwind: "$itens" }, { $group: { _id: "$itens.sku", unidades: { $sum: "$itens.qtd" }, receita: { $sum: { $multiply: ["$itens.qtd", "$itens.preco_unit"] } } } }, { $sort: { receita: -1 } } ])

    $unwind transforma cada item do array em uma entrada separada no pipeline.

    Parte 5: Exercício Prático

    Resolver consultas reais no dataset da aula

    Exercício: Catálogo e Pedidos

    Objetivo: usar MongoDB para responder perguntas sobre produtos e pedidos.

  • Liste notebooks com pelo menos 16 GB de RAM e preço abaixo de R$ 6.000
  • Mostre apenas nome, preco, specs.ram_gb e estoque
  • Reduza o estoque de um produto vendido em 1 unidade
  • Crie um índice que ajude a primeira consulta
  • Calcule a receita total por SKU usando aggregation
    •

    Template de Consulta

    db.produtos.find( { categoria: "notebook", "specs.ram_gb": { $gte: 16 }, preco: { $lt: 6000 } }, { _id: 0, nome: 1, preco: 1, "specs.ram_gb": 1, estoque: 1 } )

    Template de Índice

    db.produtos.createIndex({ categoria: 1, "specs.ram_gb": 1, preco: 1 }) db.produtos.find({ categoria: "notebook", "specs.ram_gb": { $gte: 16 }, preco: { $lt: 6000 } }).explain("executionStats")

    Template de Agregação

    db.pedidos.aggregate([ { $unwind: "$itens" }, { $group: { _id: "$itens.sku", unidades: { $sum: "$itens.qtd" }, receita: { $sum: { $multiply: ["$itens.qtd", "$itens.preco_unit"] } } } }, { $sort: { receita: -1 } } ])

    Resumo da Aula

  • MongoDB organiza dados em documentos BSON dentro de coleções
  • Consultas usam objetos de filtro, projeções e operadores como $gte, $in e $or
  • Campos aninhados e arrays são consultados naturalmente com dot notation
  • Índices continuam essenciais para consultas rápidas
  • Aggregation pipeline permite filtrar, transformar e agrupar documentos
  • Schema flexível ajuda no início, mas precisa de disciplina na aplicação
