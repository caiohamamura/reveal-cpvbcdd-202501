1. Renomeando (alias): AS

Aspas duplas não são string aqui — servem para nomes de colunas ou tabelas com espaço, maiúsculas ou acento.

SELECT nome AS "Nome do Aluno" FROM aluno;

2. Ordenando

3. Limitando os resultados

4. Funções de agregação

SELECT COUNT(*) AS total_alunos, AVG(idade) AS media_idade, AVG(nota) AS media_nota FROM aluno;

5. Agregando em grupos

SELECT idade, COUNT(*) AS n_alunos_idade, AVG(nota) AS media_nota FROM aluno GROUP BY idade ORDER BY idade DESC;

5.a. Filtrando grupos com HAVING

Apenas grupos com mais de 2 alunos:

SELECT idade, COUNT(*) AS n_alunos_idade, AVG(nota) AS media_nota FROM aluno GROUP BY idade HAVING COUNT(*) > 2 ORDER BY idade DESC;

6. Subconsultas

Alunos com nota abaixo da média:

SELECT nome, nota FROM aluno WHERE nota < (SELECT AVG(nota) FROM aluno);

7. Funções de Janela: OVER

Comparando notas com a média geral:

SELECT nome, nota, AVG(nota) OVER () AS nota_media FROM aluno ORDER BY nota DESC;

7. Funções de Janela: PARTITION BY

Rank de notas por idade:

SELECT nome, idade, nota, RANK() OVER ( PARTITION BY idade ORDER BY nota DESC) AS rank FROM aluno ORDER BY idade, nota DESC;

8. Expressões de tabela comuns (CTE): WITH

  • Subconsultas podem ficar bem complexas
  • Podem precisar ser reutilizadas várias vezes
  • CTEs permitem "gravar" a consulta temporariamente e reutilizá-la
  • CTEs deixam consultas complexas mais legíveis
    •

    8. CTE: Antes e Depois

    Sem CTE

    SELECT nome, nota, (SELECT AVG(nota) FROM aluno ) AS media_nota FROM aluno WHERE nota > ( SELECT AVG(nota) FROM aluno );

    Com CTE

    WITH aluno_media_geral AS ( SELECT nome, nota, AVG(nota) OVER () AS media_geral FROM aluno ) SELECT * FROM aluno_media_geral WHERE nota > media_geral;

    8. CTE: Resultado

    WITH aluno_media_geral AS ( SELECT nome, nota, AVG(nota) OVER () AS media_geral FROM aluno ) SELECT * FROM aluno_media_geral WHERE nota > media_geral;

    9. If-else: CASE WHEN

    SELECT nome, nota, CASE WHEN nota >= 6 THEN 'Aprovado' ELSE 'Reprovado' END AS status FROM aluno;

    10. JOIN: juntando tabelas

    Chaves primárias e estrangeiras

    Por que usamos números em vez de nomes?

    Por que usar números?

    PROBLEMA 1: Se o usuário alterar o nome, preciso atualizar em todos os lugares!

    PROBLEMA 2: Espaço em disco — cada caractere gasta 4 bytes. "Superman" = 32 bytes vs Inteiro = 4 bytes.

    PROBLEMA 3: Busca — números sequenciais facilitam encontrar a linha sem ler uma por uma.

    Produto cartesiano

    Juntando com vírgula sem condição:

    SELECT * FROM post, usuario;

    Não é bem isso que eu queria! Cada linha de post é repetida para cada usuário.

    Adicionando a condição WHERE

    SELECT * FROM post, usuario WHERE post.id_usuario = usuario.id_usuario;

    Usando aliases nas tabelas

    SELECT * FROM post p, usuario u WHERE p.id_usuario = u.id_usuario;

    Normalmente criamos aliases (apelidos) para as tabelas. Pode usar AS, mas não precisa!

    Sintaxe JOIN (recomendada)

    SELECT * FROM post p JOIN usuario u ON p.id_usuario = u.id_usuario;

    É a mesma coisa, mas deixa claro que existe uma relação entre as duas tabelas.

    11. EXISTS

    EXISTS vs IN

    Testa a existência de um campo em outra tabela na cláusula WHERE. Geralmente é mais rápido que testar com IN.

    EXISTS

    SELECT nome, idade FROM aluno a WHERE EXISTS ( SELECT 1 FROM matricula m WHERE m.id_aluno = a.id_aluno AND m.nome_da_aula = 'Matemática' );

    IN

    SELECT nome, idade FROM aluno a WHERE id_aluno IN ( SELECT id_aluno FROM matricula m WHERE m.id_aluno = a.id_aluno AND m.nome_da_aula = 'Matemática' );

    Alternativa com JOIN

    SELECT nome, idade FROM aluno a JOIN matricula m ON a.id_aluno = m.id_aluno WHERE m.nome_da_aula = 'Matemática';

    Por que usar EXISTS?

    É mais útil quando queremos algo que NÃO EXISTS — nesse caso o JOIN não resolveria.

    Exemplo: alunos que NÃO estão matriculados em Matemática

    NOT EXISTS (correto)

    SELECT nome, idade FROM aluno a WHERE NOT EXISTS ( SELECT 1 FROM matricula m WHERE m.id_aluno = a.id_aluno AND m.nome_da_aula = 'Matemática' );

    != com IN (errado!)

    SELECT nome, idade FROM aluno a WHERE id_aluno IN ( SELECT id_aluno FROM matricula m WHERE m.nome_da_aula != 'Matemática' );

    Isso retorna alunos matriculados em outras matérias, não os que NÃO estão em Matemática!
