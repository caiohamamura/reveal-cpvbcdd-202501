    DO Blocks — Blocos Anônimos

    • Bloco de código sem nome, executado uma vez
    • Não é armazenado no banco
    • Útil para migrações e testes rápidos
    • Suportado nativamente no PostgreSQL
    • Permite criar variáveis
    Estrutura: 
                    DO $$
DECLARE
  -- variáveis
BEGIN
  -- lógica
  RAISE NOTICE '...';
END $$;

    O que são os $$ ?

    • Dollar quoting — alternativa a aspas simples para strings
    • Evita escapar ' dentro de strings grandes
    • Pode ter tag: $body$...$body$, $func$...$func$
    • $$ = tag vazia — o mais comum

    Sem dollar quoting:

    SELECT 'It''s a ''test''';

    Com dollar quoting:

    SELECT $$It's a 'test'$$;

    Em DO blocks e functions, $$ delimita o corpo do código PL/pgSQL

    DO Block — Olá, PostgreSQL!

    DO $$ BEGIN RAISE NOTICE 'Olá, Banco de Dados II!'; END $$;

    Resultado:

    NOTICE:  Olá, Banco de Dados II!

    DO Block — Variáveis e Lógica

    DO $$ DECLARE nome_aluno VARCHAR(50) := 'ADS 3º'; total_clientes INTEGER; BEGIN SELECT COUNT(*) INTO total_clientes FROM clientes; IF total_clientes > 10 THEN RAISE NOTICE 'Turma % tem % clientes!', nome_aluno, total_clientes; ELSE RAISE NOTICE 'Turma % — poucos clientes ainda', nome_aluno; END IF; END $$;

    DO Block — Exemplo Prático

    Migrar emails para minúsculo e contar atualizações

    DO $$ DECLARE count_updated INTEGER; BEGIN UPDATE clientes SET email = LOWER(email) WHERE email != LOWER(email); GET DIAGNOSTICS count_updated = ROW_COUNT; RAISE NOTICE 'E-mails normalizados: %', count_updated; END $$;

    DO block = execução imediata, sem reuso, sem parâmetros de saída

    Quando usar DO Blocks?

  • Script de migração único
  • Atualização massiva de dados sem reaproveitamento
  • Teste rápido de lógica PL/pgSQL
  • Automação de tarefas administrativas (roles, schemas)
    •

    Stored Procedures — O que são?

  • Objeto armazenado, não retorna valor
  • Chamada via CALL
  • Pode fazer COMMIT/ROLLBACK interno
  • Ideal para operações de mutação
    •  
    CREATE PROCEDURE nome(pars)
AS $$
BEGIN
    -- lógica de mutação
END;
$$ LANGUAGE plpgsql;

-- Chamada:
CALL nome(1, 2);

    Procedure — Inserir Cliente

    CREATE OR REPLACE PROCEDURE sp_novo_cliente( p_nome VARCHAR, p_email VARCHAR, p_cidade VARCHAR, p_limite DECIMAL(10,2) ) AS $$ BEGIN INSERT INTO clientes (nome, email, cidade, limite_credito) VALUES (p_nome, p_email, p_cidade, p_limite); RAISE NOTICE 'Cliente % inserido com sucesso!', p_nome; END; $$ LANGUAGE plpgsql; 
    CALL sp_novo_cliente('Eduardo Lima', 'eduardo@email.com', 'São Paulo', 4500);

    Procedure — Validação e Exceção

    CREATE OR REPLACE PROCEDURE sp_atualizar_status( p_pedido_id INTEGER, p_novo_status VARCHAR ) AS $$ BEGIN IF NOT EXISTS ( SELECT 1 FROM pedidos WHERE id = p_pedido_id ) THEN RAISE EXCEPTION 'Pedido #% não encontrado!', p_pedido_id; END IF; UPDATE pedidos SET status = p_novo_status WHERE id = p_pedido_id; RAISE NOTICE 'Pedido #% → %', p_pedido_id, p_novo_status; END; $$ LANGUAGE plpgsql;

    Procedure — Transferência de Estoque

    CREATE OR REPLACE PROCEDURE sp_transferir_estoque( p_origem INTEGER, p_destino INTEGER, p_qtd INTEGER ) AS $$ DECLARE v_estoque INTEGER; BEGIN SELECT estoque INTO v_estoque FROM produtos WHERE id = p_origem; IF v_estoque < p_qtd THEN RAISE EXCEPTION 'Estoque insuficiente! Disponível: %', v_estoque; END IF; UPDATE produtos SET estoque = estoque - p_qtd WHERE id = p_origem; UPDATE produtos SET estoque = estoque + p_qtd WHERE id = p_destino; RAISE NOTICE 'Transferidos % itens: #% → #%', p_qtd, p_origem, p_destino; END; $$ LANGUAGE plpgsql;

    Quando usar Procedures?

    • Operações de mutação (INSERT, UPDATE, DELETE)
    • Processos em lote que precisam de transações
    • Quando precisa usar COMMIT/ROLLBACK interno
    • Processos que alteram múltiplas tabelas atomicamente
    • Tarefas de importação ou migração de dados

    Functions — O que são?

  • Objeto armazenado no banco
  • Recebe parâmetros e retorna um valor
  • Pode ser usada em SELECT, WHERE, expressions
  • Reutilizável em múltiplos lugares
    •  
    CREATE FUNCTION nome(pars)
RETURNS tipo AS $$
BEGIN
    RETURN valor;
END;
$$ LANGUAGE plpgsql;

    Function Escalar — Soma

    CREATE OR REPLACE FUNCTION somar( a INTEGER, b INTEGER ) RETURNS INTEGER AS $$ BEGIN RETURN a + b; END; $$ LANGUAGE plpgsql; 
    SELECT somar(3, 5);  -- resultado: 8

    Function — Calculadora de Comissão

    CREATE OR REPLACE FUNCTION calcular_comissao( p_venda NUMERIC(10,2), p_taxa NUMERIC(3,2) DEFAULT 0.05 ) RETURNS NUMERIC(10,2) AS $$ BEGIN IF p_venda < 0 THEN RAISE EXCEPTION 'Venda não pode ser negativa: %', p_venda; END IF; RETURN p_venda * p_taxa; END; $$ LANGUAGE plpgsql STRICT; 
    SELECT calcular_comissao(1000.00);  -- 50.00

    Tipos de Retorno

    TipoDescriçãoExemplo de uso
    RETURNS tipoEscalarINTEGER, TEXT, BOOLEAN
    RETURNS TABLE(...)Tabela virtualRelatórios, filtros
    RETURNS SETOF tabelaConjunto de linhasListagens

    Function que Retorna Tabela

    CREATE OR REPLACE FUNCTION fn_produtos_por_categoria( p_categoria VARCHAR ) RETURNS TABLE( id INTEGER, nome VARCHAR, preco DECIMAL(10,2) ) AS $$ BEGIN RETURN QUERY SELECT pro.id, pro.nome, pro.preco FROM produtos pro WHERE pro.categoria = p_categoria; END; $$ LANGUAGE plpgsql; 
    SELECT * FROM fn_produtos_por_categoria('Eletrônicos');

    Modos de Parâmetros

  • IN — recebe valor (padrão)
  • OUT — retorna valor pela assinatura
  • INOUT — entrada e saída
  • VARIADIC — número variável de args
    •
    -- OUT: retorna sem RETURN CREATE FUNCTION dividir( a NUMERIC, b NUMERIC, OUT resultado NUMERIC, OUT resto NUMERIC ) AS $$ BEGIN resultado := a / b; resto := a % b; END; $$ LANGUAGE plpgsql;

    Function — Estatísticas com OUT

    CREATE OR REPLACE FUNCTION fn_estatisticas_cliente( p_cliente_id INTEGER, OUT total_pedidos INTEGER, OUT valor_gasto DECIMAL(10,2), OUT total_itens INTEGER ) AS $$ BEGIN SELECT COUNT(*), COALESCE(SUM(valor_total), 0) INTO total_pedidos, valor_gasto FROM pedidos WHERE cliente_id = p_cliente_id; SELECT COALESCE(SUM(pi.quantidade), 0) INTO total_itens FROM pedidos p JOIN pedidos_itens pi ON p.id = pi.pedido_id WHERE p.cliente_id = p_cliente_id; END; $$ LANGUAGE plpgsql; 
    SELECT * FROM fn_estatisticas_cliente(1);

    INOUT — Entrada e Saída

    -- INOUT: recebe, modifica e retorna CREATE FUNCTION dobro_inout( INOUT valor NUMERIC ) AS $$ BEGIN valor := valor * 2; END; $$ LANGUAGE plpgsql; 
    SELECT dobro_inout(5);  -- resultado: 10

    VARIADIC — Múltiplos Argumentos

    -- VARIADIC: número variável de argumentos CREATE FUNCTION somar_varios( VARIADIC nums NUMERIC[] ) RETURNS NUMERIC AS $$ DECLARE total NUMERIC := 0; BEGIN FOR i IN 1..array_length(nums, 1) LOOP total := total + nums[i]; END LOOP; RETURN total; END; $$ LANGUAGE plpgsql; 
    SELECT somar_varios(1, 2, 3, 4, 5);  -- resultado: 15

    Quando usar Functions?

  • Reutilização em SELECT, WHERE, CHECK constraints
  • Cálculos recorrentes (comissão, imposto, frete)
  • Validação de dados em constraints
  • Geração de relatórios complexos (RETURNS TABLE)
  • Composable — pode ser usada dentro de outras queries
    •

    Resumo da Aula

  • DO Block — executa uma vez, sem nome, sem reuso
  • Procedure — mutação de estado, CALL, transações
  • Function — retorna valor, composável em SQL
  • Escolha baseada em reuso e retorno
