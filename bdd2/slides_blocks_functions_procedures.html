🔧 DO Blocks — Blocos Anônimos

  • Blocos de código sem nome, executados uma vez
  • Úteis para scripts de migração e testes rápidos
  • Não armazenados no banco — executa e pronto
  • Suportado nativamente no PostgreSQL
💡 Script de uso único

DO Block — Exemplo Prático


DO $$
DECLARE
  total INTEGER;
BEGIN
  SELECT COUNT(*) INTO total FROM customers;
  RAISE NOTICE 'Total de clientes: %', total;
END $$;

⚡ Functions — Funções Armazenadas

  • Blocos de código armazenados no banco
  • Podem ser reutilizados em múltiplas chamadas
  • Retornam valores (escalares ou tabelas)
  • Podem receber parâmetros de entrada
  • Suportam LANGUAGE SQL ou LANGUAGE plpgsql
📦 Código reutilizável

Function Escalar — Exemplo


CREATE OR REPLACE FUNCTION total_pedidos(p_customer_id INT)
RETURNS NUMERIC(10,2) AS $$
DECLARE
  total NUMERIC(10,2);
BEGIN
  SELECT COALESCE(SUM(quantity * unit_price), 0)
  INTO total
  FROM order_details
  WHERE customer_id = p_customer_id;
  
  RETURN total;
END;
$$ LANGUAGE plpgsql;

Function Escalar — Uso


-- Chamada da função
SELECT total_pedidos(1);

-- Uso em consulta
SELECT company_name, total_pedidos(customer_id) AS total
FROM customers
ORDER BY total DESC;

Funções escalares retornam um valor único.
Podem ser usadas em SELECT, WHERE, ORDER BY, etc.

Function Tabelar — Retorna Tabela


CREATE OR REPLACE FUNCTION clientes_por_cidade(p_city VARCHAR(50))
RETURNS TABLE(
  customer_id INT,
  company_name VARCHAR(100),
  city VARCHAR(50)
) AS $$
BEGIN
  RETURN QUERY
    SELECT customer_id, company_name, city
    FROM customers
    WHERE city ILIKE '%' || p_city || '%';
END;
$$ LANGUAGE plpgsql;

Function Tabelar — Uso


-- Chamada: treated like a table
SELECT * FROM clientes_por_cidade('London');

-- Junto com outras tabelas
SELECT c.company_name, p.product_name
FROM clientes_por_cidade('London') c
JOIN orders o ON c.customer_id = o.customer_id
JOIN order_details od ON o.order_id = od.order_id
JOIN products p ON od.product_id = p.product_id;

RETURN TABLE permite usar a função como se fosse uma tabela real!

📋 Stored Procedures

  • Semelhante a Functions, mas com diferenças-chave
  • Procedures não retornam valores diretamente
  • Suportam transações (COMMIT/ROLLBACK)
  • Chamadas com CALL, não SELECT
  • Ideais para operações de escrita complexa
🔄 Transações completas

Procedure — Exemplo Completo


CREATE OR REPLACE PROCEDURE atualizar_estoque(
  p_product_id INT,
  p_quantidade INT,
  p_tipo_mov CHAR(1) -- 'E' = entrada, 'S' = saída
) AS $$
BEGIN
  IF p_tipo_mov = 'E' THEN
    UPDATE products
    SET units_in_stock = units_in_stock + p_quantidade
    WHERE product_id = p_product_id;
  ELSIF p_tipo_mov = 'S' THEN
    UPDATE products
    SET units_in_stock = units_in_stock - p_quantidade
    WHERE product_id = p_product_id;
  END IF;
  
  COMMIT;
END;
$$ LANGUAGE plpgsql;

Procedure — Chamada


-- Entrada de 50 unidades
CALL atualizar_estoque(1, 50, 'E');

-- Saída de 10 unidades
CALL atualizar_estoque(1, 10, 'S');

-- Verificar resultado
SELECT product_name, units_in_stock
FROM products
WHERE product_id = 1;

CALL invoca a procedure. Transactions garantem integridade!

Comparando as Três Abordagens

🔧 DO Block

  • Executa uma vez
  • Sem nome
  • Não armazenado
  • Migrações, testes

⚡ Function

  • Reutilizável
  • Retorna valor
  • Em SELECT/WHERE
  • Cálculos, consultas

📋 Procedure

  • Reutilizável
  • Sem retorno direto
  • Transações
  • Escrita, atualização
