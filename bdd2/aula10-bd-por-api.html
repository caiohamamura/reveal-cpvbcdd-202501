    Problema

  • Qualquer um poderia tentar descobrir o usuário e senha e apagar ou ter acesso aos dados

    Expondo por meio de uma API

  • Uma API é um serviço intermediário
  • Vai fazer o meio de campo entre a web e o banco de dados
  • VANTAGENS
  • Permite criar um mecanismo de autenticação intermediário e só envia para o BD requisições autenticadas
  • Permite definir funções intermediárias que podem fazer várias operações no BD
  • Pode esconder o IP do servidor de banco de dados (você pode rodar essa aplicação em outro servidor)
    •

    PostGraphile

    Gerador automático de API tipo GraphQL

  • Padrão criado pelo Facebook em 2012
  • O GraphQL é uma maneira de definir consultas pelo cliente de forma mais amigável
  • Solicita de maneira parecida com o resultado que retornará em JSON
    •

    JSON: JavaScript Object Notation

  • Notação de objeto do javascript é o padrão de transferência de dados na web
  • Ele é legível para humanos, além de ser eficiente

    Instalando e rodando o PostGraphile

    Banco de Dados

    Requisitos

  • Instalação do Node
  • https://nodejs.org/dist/v22.11.0/node-v22.11.0-x64.msi
    •

    Após instalar o node

  • Abrir cmd
  • npm install -g postgraphile
  • No linux: sudo npm install -g postgraphile
  • Fechar e abrir novamente o cmd
    •

    Rodando o PostGraphile

    O PostGraphile vai se conectar ao SGBD, então vamos rodar o comando no cmd

    postgraphile -c postgres://<usuario>:<senha>@<endereco>/<nome_banco> --enhance-graphiql --cors --watch

    Rodando o PostGraphile

  • -c: define onde vai se conectar, troque <endereco> pelo endereço da máquina, se for local, localhost
  • --enhance-graphiql: vai criar uma interface de consulta com ajuda
  • --cors: vai permitir fazer a consulta de qualquer site (caso contrário precisa ser do mesmo endereço de origem)
  • --watch: vai verificar mudanças no banco e atualizar automaticamente o GraphQL
    •

    postgraphile -c postgres://<endereco>/<nome_banco> --enhance-graphiql --cors --watch
