📸 Materialized Views como Snapshot

Congelando dados no tempo

Banco de Dados 2 — IFSP Capivari

O Problema

Imagine que a diretoria pediu um relatório de vendas de março.

Você roda a query no dia 5 de abril.

Mas entre 31/03 e 05/04...

  • Pedidos foram cancelados
  • Valores foram alterados
  • Produtos foram removidos

O relatório não mostra mais os dados reais de março.

View Regular Não Resolve


-- Uma view regular SEMPRE re-executa a query
CREATE VIEW vw_vendas_marco AS
SELECT produto, SUM(quantidade) AS total
FROM pedidos
WHERE data_pedido BETWEEN '2026-03-01' AND '2026-03-31'
GROUP BY produto;

Cada SELECT nessa view reflete o estado atual das tabelas, não o estado do passado.

Se um pedido foi deletado em abril, ele desaparece do resultado.

Câmera ao vivo

Materialized view = fotografia que preserva o momento exato.

View regular = câmera ao vivo que mostra o estado atual.

Criando um Snapshot


CREATE MATERIALIZED VIEW mv_snapshot_vendas_marco AS
SELECT 
    p.produto,
    SUM(ip.quantidade) AS total_vendido,
    SUM(ip.quantidade * ip.preco_unitario) AS valor_total,
    NOW() AS snapshot_em
FROM itens_pedido ip
JOIN pedidos p ON ip.id_pedido = p.id_pedido
WHERE p.data_pedido BETWEEN '2026-03-01' AND '2026-03-31'
GROUP BY p.produto
WITH DATA;

A coluna snapshot_em registra quando a foto foi tirada.

⚠️ Regra de Ouro do Snapshot


-- NUNCA faça isso em um snapshot:
REFRESH MATERIALIZED VIEW mv_snapshot_vendas_marco;

REFRESH destrói o snapshot e tira uma nova foto.

Os dados originais de março seriam substituídos pelo estado atual.

Um snapshot que recebeu refresh não é mais um snapshot.

Quando Usar Snapshot?

  • 📊 Fechamento contábil — balancete mensal que não pode mudar
  • 📋 Auditoria — evidência do estado dos dados em uma data
  • 📈 Relatórios de desempenho — ranking congelado para comparação
  • 🧾 Compliance fiscal — dados que a lei exige preservar como estavam
  • 🔄 Baseline para comparação — comparar mês atual vs mês anterior congelado

🤔 Reflexão

"Uma empresa de e-commerce faz o fechamento do mês às 23h59 do último dia. No dia seguinte, o financeiro percebe que um estorno de R$ 15.000 foi processado retroativamente. Como garantir que o relatório de fechamento não seja afetado?"

Resposta: O snapshot (MV sem refresh) preserva os dados como estavam no momento do fechamento, independentemente do que acontecer depois.
