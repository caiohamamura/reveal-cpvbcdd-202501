    Problema central

    Como saber se um ponto é estranho?

    Depende de...

  • Contexto: o que é "normal" neste domínio?
  • Distribuição dos dados: simétrica, assimétrica, multimodal?
  • Relação entre variáveis: combinações inesperadas.
  • Custo do erro: o que dói mais — falso positivo ou falso negativo?
  • Objetivo do problema: detecção, investigação ou limpeza?

    Isolation Forest — intuição

    Pontos anômalos são mais fáceis de isolar com cortes aleatórios.
    Ponto isolado → poucos cortes Ponto normal → muitos cortes

    Isolation Tree (iTree)

  • Cada árvore é construída com uma amostra aleatória dos dados (subamostragem).
  • Em cada nó, escolhe-se uma variável aleatória \(X_j\) e um ponto de corte aleatório \(p\) entre \(\min(X_j)\) e \(\max(X_j)\).
  • Se \(x_{j} < p\) → vai para a subárvore esquerda; senão → direita.
  • O processo repete até o ponto ficar isolado (folha) ou atingir a altura máxima.
    •

    Comprimento do caminho \(h(x)\)

    Para cada ponto \(x\), medimos quantos cortes foram necessários para isolá-lo em uma árvore:

    \(h(x)\) = número de arestas da raiz até a folha que contém \(x\)

  • Anomalias → \(h(x)\) pequeno (poucos cortes).
  • Pontos normais → \(h(x)\) grande (muitos cortes).
    •

    Esperança do caminho \(E(h(x))\)

    Como uma única árvore é ruidosa, usamos a média sobre \(t\) árvores:

    \(E(h(x)) = \frac{1}{t} \sum_{i=1}^{t} h_i(x)\)

    onde \(h_i(x)\) é o comprimento do caminho de \(x\) na árvore \(i\).

    Fator de normalização \(c(n)\)

    O caminho esperado depende do tamanho da subamostra \(n\). Para busca binária aleatória:

    \(c(n) = 2H(n-1) - \frac{2(n-1)}{n}\)

    onde \(H(i) \approx \ln(i) + \gamma\) é o \(i\)-ésimo número harmônico e \(\gamma \approx 0,57721\) é a constante de Euler-Mascheroni.

    Score de anomalia \(s(x, n)\)

    \(s(x, n) = 2^{\large -\frac{E(h(x))}{c(n)}}\)

    CondiçãoInterpretação
    \(s(x, n) \to 1\)ponto é anômalo (\(E(h(x)) \ll c(n)\))
    \(s(x, n) \approx 0,5\)ponto é normal (\(E(h(x)) \approx c(n)\))
    \(s(x, n) \to 0\)ponto é muito normal (\(E(h(x)) \gg c(n)\))

    Passo a passo do algoritmo

  • 1. Subamostrar: escolher \(\psi\) pontos aleatórios (típico: \(\psi = 256\)).
  • 2. Para cada árvore \(i = 1 \ldots t\): construir iTree com a subamostra.
  • 3. Para cada ponto \(x\): calcular \(h_i(x)\) em cada árvore.
  • 4. Calcular \(E(h(x)) = \frac{1}{t}\sum h_i(x)\).
  • 5. Calcular \(s(x, n) = 2^{-E(h(x))/c(n)}\).
  • 6. Classificar como anomalia se \(s(x, n) > \tau\) (típico: \(\tau = 0,5\)).
    •

    contamination=0.05 → espera 5% de anomalias nos dados.

    Local Outlier Factor (LOF) — intuição

    Um ponto é suspeito se está em uma região muito menos densa que seus vizinhos.

    LOF: \(k\)-distância e vizinhança

    Para cada ponto \(p\), definimos:

    \(k\text{-distância}(p)\) = distância ao \(k\)-ésimo vizinho mais próximo

    \(N_k(p)\) = conjunto dos \(k\) vizinhos mais próximos de \(p\)

    Todos os pontos \(o\) com \(d(p, o) \leq k\text{-distância}(p)\) pertencem a \(N_k(p)\).

    LOF: distância de reachabilidade

    A reachability distance de \(p\) em relação a \(o\) é:

    \(\text{reach-dist}_k(p, o) = \max\bigl\{k\text{-distância}(o),\; d(p, o)\bigr\}\)

    Isso suaviza as distâncias: pontos muito próximos de \(o\) herdam a \(k\)-distância de \(o\), evitando variações extremas.

    LOF: densidade de reachabilidade local

    A densidade local de \(p\) é o inverso da média das reachability distances:

    \(\text{LRD}_k(p) = \frac{|N_k(p)|}{\sum_{o \in N_k(p)} \text{reach-dist}_k(p, o)}\)

  • LRD alto → \(p\) está em região densa.
  • LRD baixo → \(p\) está em região esparsa.
    •

    LOF: fator de outlier local

    \(\text{LOF}_k(p) = \frac{\sum_{o \in N_k(p)} \text{LRD}_k(o)}{|N_k(p)| \cdot \text{LRD}_k(p)}\)

    Valor de LOFInterpretação
    \(\text{LOF}_k(p) \approx 1\)\(p\) tem densidade similar aos vizinhos → normal
    \(\text{LOF}_k(p) > 1\)\(p\) está em região menos densa → candidato a outlier
    \(\text{LOF}_k(p) < 1\)\(p\) está em região mais densa → ponto interno ao cluster

    LOF: passo a passo do algoritmo

  • 1. Para cada ponto \(p\): encontrar \(N_k(p)\) (k vizinhos mais próximos).
  • 2. Calcular \(k\text{-distância}(p)\) para todo \(p\).
  • 3. Calcular \(\text{reach-dist}_k(p, o)\) para todo par \((p, o)\) onde \(o \in N_k(p)\).
  • 4. Calcular \(\text{LRD}_k(p)\) para todo \(p\).
  • 5. Calcular \(\text{LOF}_k(p)\) para todo \(p\).
  • 6. Classificar como anomalia se \(\text{LOF}_k(p) > \tau\) (típico: \(\tau \approx 1,5\) a \(2\)).
    •

    LOF vs DBSCAN

    DBSCAN

    • Cria clusters por densidade.
    • Ruído = pontos fora de regiões densas.
    • Binário: pertence ou não a um cluster.

    LOF

    • Compara densidade local entre vizinhos.
    • Score contínuo: quanto maior, mais anômalo.
    • Detecta anomalias dentro de clusters.

    LOF compara densidade local; DBSCAN cria clusters e marca ruído.
