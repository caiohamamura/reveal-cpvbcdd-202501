Revisão: o que já temos na caixa de ferramentas

  • Regressão e classificação para problemas com alvo conhecido.
  • KNN, árvore de decisão e Random Forest para previsão.
  • PCA para muitas variáveis e visualização.
  • K-Means, clustering hierárquico e DBSCAN para segmentação.
  • IQR, Z-score, Isolation Forest, LOF e One-Class SVM para anomalias.
    •

    Motivação: o dataset pode vir de qualquer lugar

  • Cada grupo pode receber um dataset diferente.
  • Nem todo problema começa com "treine um modelo".
  • Às vezes a melhor resposta é uma tabela, um gráfico ou uma comparação.
  • O primeiro erro comum é escolher o algoritmo antes de entender a pergunta.
    •

    Hoje a aula é um mapa: problema → pergunta → tipo de análise → método.

    Fase 1: comece pelo problema

    O professor quer descrever, comparar, prever, agrupar, detectar exceções ou explicar uma relação?

    Algoritmo não é ponto de partida

    Jeito forte

    "Existe uma variável alvo?"

    Fase 2: procure a variável alvo

    Existe uma coluna que representa a resposta que queremos prever, explicar ou comparar?

    Regra prática: tipo do alvo

    Se o alvo é... Então provavelmente... Exemplo
    Ausente EDA, clustering, PCA, anomalias perfis de usuários
    Numérico contínuo regressão nota final, consumo, preço
    Categoria binária classificação ou regressão logística evadiu: sim/não
    Múltiplas categorias classificação curso, turno, nível de risco
    Ordenado por tempo tendência, sazonalidade, séries temporais empréstimos por mês
    Evento muito raro anomalia ou classificação desbalanceada falha de máquina

    Fase 3: dados que pedem cuidado especial

  • Séries temporais: a ordem dos registros importa.
  • Texto livre: respostas precisam virar atributos numéricos.
  • Questionários: cada pergunta pode ser variável, escala ou texto.
  • Eventos raros: a classe importante pode quase não aparecer.
    •

    Séries temporais: quando a data importa

    • Exemplo: empréstimos da biblioteca por semana.
    • O objetivo pode ser descrever tendência, comparar períodos ou prever futuro.
    • Não embaralhe treino e teste quando a ordem temporal importa.
    • Procure tendência, sazonalidade, feriados e mudanças de política.

    Métodos comuns

    linha temporal

    média móvel

    decomposição sazonal

    forecast simples

    Série temporal em Python

    import pandas as pd df = pd.read_csv("emprestimos.csv", parse_dates=["data"]) serie = ( df.set_index("data") .resample("W") .size() .rename("emprestimos") ) media_movel = serie.rolling(4).mean() serie.plot() media_movel.plot()

    Pergunta possível: os empréstimos aumentam em semanas de prova ou caem nas férias?

    Texto e respostas abertas

    • Exemplo: "O que dificultou sua permanência no curso?"
    • Texto precisa ser limpo, tokenizado e transformado em números.
    • A análise pode ser descritiva, classificação ou agrupamento de temas.
    • Leia amostras manualmente antes de confiar em métricas automáticas.

    Métodos comuns

    contagem de palavras

    TF-IDF

    classificação de texto

    clusterização de respostas

    Texto em Python: TF-IDF

    import pandas as pd from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.cluster import KMeans df = pd.read_csv("respostas.csv") textos = df["resposta_aberta"].fillna("") vetor = TfidfVectorizer( lowercase=True, stop_words=["de", "da", "do", "e", "a", "o", "para"] ) X = vetor.fit_transform(textos) clusters = KMeans(n_clusters=4, random_state=42).fit_predict(X) df["tema_estimado"] = clusters

    Depois do agrupamento, leia exemplos de cada cluster e dê nomes interpretáveis aos temas.

    Questionários: nem tudo é texto livre

    Tipo de pergunta Como analisar Exemplo
    Escala Likert frequência, mediana, comparação por grupo satisfação de 1 a 5
    Múltipla escolha tabelas, barras, associação entre categorias principal meio de transporte
    Muitas perguntas relacionadas PCA, consistência, criação de índice engajamento acadêmico
    Resposta aberta TF-IDF, temas, classificação manual ou automática sugestões para o curso

    Questionário em Python

    import pandas as pd df = pd.read_csv("questionario.csv") itens = ["q1_satisfacao", "q2_apoio", "q3_estrutura", "q4_clareza"] resumo = df[itens].agg(["mean", "median", "std"]).T por_curso = df.groupby("curso")[itens].mean() df["indice_experiencia"] = df[itens].mean(axis=1) df.boxplot(column="indice_experiencia", by="curso")

    Uma boa conclusão não é "usei Python"; é dizer qual grupo, tema ou item precisa de atenção.

    Frases boas para projeto final

    • "A pergunta principal é prever Y a partir de X."
    • "Como não existe variável alvo, começamos com EDA e segmentação."
    • "Como o alvo é binário, tratamos como classificação."
    • "Como o evento é raro, avaliamos recall e falsos negativos com cuidado."
    • "Como há ordem temporal, separamos treino e teste por data."
    • "Como há muitas variáveis correlacionadas, usamos PCA para visualização."

    Fase 5: atividade prática

    Objetivo: classificar cartões de projeto antes de escrever qualquer código.

  • Identifique a pergunta principal.
  • Decida se existe variável alvo.
  • Classifique o alvo: numérico, categórico, temporal, raro ou ausente.
  • Escolha métodos do curso que poderiam aplicar.
  • Escolha o primeiro gráfico útil.
  • Escreva uma conclusão final possível.

    Cartão 1: biblioteca

    Dataset

    Registros de empréstimos: data, livro, categoria, curso do aluno, atraso, renovação.

    Pedido do professor

    "Quero entender quais materiais são mais usados e se existem perfis de uso por curso."

    Entregável esperado

    Gráficos claros e uma recomendação para compra ou redistribuição do acervo.

    Cartão 2: evasão

    Dataset

    Histórico de estudantes: frequência, notas parciais, idade, curso, turno, status final.

    Pedido do professor

    "Quero identificar alunos com maior risco de evasão antes do fim do semestre."

    Entregável esperado

    Modelo interpretável, variáveis mais importantes e cuidado com falsos negativos.

    Cartão 3: manutenção

    Dataset

    Medições de máquinas: temperatura, vibração, corrente, data, operador, registro de falha.

    Pedido do professor

    "Quero saber quais leituras parecem fora do padrão e se isso antecipa falhas."

    Entregável esperado

    Lista priorizada de registros suspeitos, gráfico temporal e proposta de monitoramento.

    Resumo da Aula

  • Comece pela pergunta, não pelo algoritmo.
  • O tipo da variável alvo indica o caminho inicial.
  • Nem todo dataset pede machine learning supervisionado.
  • Séries temporais, texto e questionários exigem preparação específica.
  • Um bom projeto termina com uma conclusão útil para o problema real.
