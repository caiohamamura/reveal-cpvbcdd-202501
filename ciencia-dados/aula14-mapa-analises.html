Revisão: o que já temos na caixa de ferramentas

  1. Inferência estatística.
  2. Regressão e classificação para problemas com alvo conhecido.
  3. KNN, árvore de decisão e Random Forest para previsão.
  4. PCA para muitas variáveis e visualização.
  5. K-Means, clustering hierárquico e DBSCAN para segmentação.
  6. IQR, Z-score, Isolation Forest, LOF e One-Class SVM para anomalias.

Trabalho Final: o dataset pode vir de qualquer lugar

  • Cada grupo vai receber um dataset diferente (por enquanto não temos muitos).
  • Nem todo problema começa com "treine um modelo".
  • Às vezes a melhor resposta é uma tabela, um gráfico ou uma comparação.
  • O primeiro erro comum é escolher o algoritmo antes de entender a pergunta.

    Existe uma coluna que representa a resposta que queremos prever, explicar ou comparar?

    Fase 3: dados que pedem cuidado especial

  • Séries temporais: a ordem dos registros importa.
  • Texto livre: respostas precisam virar atributos numéricos.
  • Questionários: cada pergunta pode ser variável, escala ou texto.
  • Eventos raros: a classe importante pode quase não aparecer.

    Séries temporais: quando a data importa

    • Exemplo: empréstimos da biblioteca por semana.
    • O objetivo pode ser descrever tendência, comparar períodos ou prever futuro.
    • Não embaralhe treino e teste quando a ordem temporal importa.
    • Procure tendência, sazonalidade, feriados e mudanças de política.

    Métodos comuns

    linha temporal

    média móvel

    decomposição sazonal

    forecast simples

    Exemplo Plotly: empréstimos por semana

    Como interpretar

    • Tendência: alta até março, queda em abril, retomada em maio.
    • Barras positivas: demanda acima do esperado pela tendência.
    • Barras negativas: semanas fracas além da queda normal.
    • Uso prático: reforçar atendimento em provas e ajustar prazos no recesso.

    Passo 1: suavizar ruído com média móvel.

    Passo 2: decompor tendência e padrão sazonal.

    Resultado: linha verde = demanda esperada; barras = semanas acima ou abaixo dela. Interpretação: provas elevam a demanda; recesso/feriados derrubam além da tendência.

    Passo 3: projetar um forecast simples para as próximas semanas.

    Pergunta: semanas de prova aumentam o uso da biblioteca? Férias reduzem?

    Série temporal em Python

    import pandas as pd import plotly.graph_objects as go df = pd.read_csv("emprestimos.csv", parse_dates=["data"]) serie = ( df.set_index("data") .resample("W") .size() .rename("emprestimos") ) media_movel = serie.rolling(4).mean() fig = go.Figure() fig.add_scatter( x=serie.index, y=serie, mode="lines+markers", name="empréstimos" ) fig.add_scatter( x=media_movel.index, y=media_movel, mode="lines", name="média móvel de 4 semanas" ) fig.update_layout( title="Empréstimos por semana", xaxis_title="semana", yaxis_title="quantidade" ) fig.show()

    A agregação semanal precisa fazer sentido para o dado. Se a meta for prever, separe treino e teste por data.

    Decomposição e forecast em Python

    from statsmodels.tsa.seasonal import seasonal_decompose decomp = seasonal_decompose( serie, model="additive", period=4 ) ultimas = serie.tail(4) passo = (ultimas.iloc[-1] - ultimas.iloc[0]) / 3 forecast = [ serie.iloc[-1] + passo * i for i in range(1, 5) ] # Interpretação: # tendência: direção geral # sazonal: padrão que se repete # forecast: hipótese simples, não certeza

    Interpretação: tendência explica a direção geral; sazonalidade mostra semanas acima/abaixo do esperado; forecast é hipótese de planejamento, não certeza.

    Resultado da execução

    Tendência: sobe até cerca de 89,6 empréstimos/semana em março, cai para 48,0 no fim de abril e volta a 76,9 em maio.

    Sazonal: o ciclo de 4 semanas fica em torno de [-5,8; -2,7; +4,2; +4,3]. As duas últimas posições do ciclo tendem a ficar acima da tendência.

    Forecast simples: usando as últimas quatro semanas [68, 84, 91, 77], o passo é +3 por semana e a projeção fica [80, 83, 86, 89].

    Leitura: abril foi fraco além da tendência; maio recupera. A previsão sugere retomada, mas ainda precisa ser confrontada com calendário acadêmico.

    Texto em Python: TF-IDF + clusters

    import pandas as pd import plotly.express as px from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.cluster import KMeans from sklearn.decomposition import TruncatedSVD df = pd.read_csv("respostas.csv") textos = df["resposta_aberta"].fillna("") vetor = TfidfVectorizer( lowercase=True, stop_words=["de", "da", "do", "e", "a", "o", "para"] ) X = vetor.fit_transform(textos) clusters = KMeans( n_clusters=4, random_state=42, n_init="auto" ).fit_predict(X) coords = TruncatedSVD(n_components=2, random_state=42).fit_transform(X) df["cluster"] = clusters.astype(str) df["x"] = coords[:, 0] df["y"] = coords[:, 1] fig = px.scatter( df, x="x", y="y", color="cluster", hover_data=["resposta_aberta"] ) fig.show()

    A posição 2D ajuda a visualizar. Os nomes dos clusters vêm das palavras fortes e da leitura de exemplos reais.

    Frases boas para projeto final

    • "A pergunta principal é prever Y a partir de X."
    • "Como não existe variável alvo, começamos com EDA e segmentação."
    • "Como o alvo é binário, tratamos como classificação."
    • "Como o evento é raro, avaliamos recall e falsos negativos com cuidado."
    • "Como há ordem temporal, separamos treino e teste por data."
    • "Como há muitas variáveis correlacionadas, usamos PCA para visualização."

    Resumo da Aula

  • Comece pela pergunta, não pelo algoritmo.
  • O tipo da variável alvo indica o caminho inicial.
  • Nem todo dataset pede machine learning supervisionado.
  • Séries temporais, texto e questionários exigem preparação específica.
  • Um bom projeto termina com uma conclusão útil para o problema real.
