Trabalho Final: o dataset pode vir de qualquer lugar

  • Cada grupo vai receber um dataset diferente (por enquanto não temos muitos).

  • O primeiro erro comum é escolher o algoritmo antes de entender a pergunta.

    Intuição: meses como dummies

    Primeiro pense como uma regressão linear comum:

    • criamos uma variável binária para cada mês;
    • éFevereiro = 1 quando a data está em fevereiro;
    • éFevereiro = 0 nos outros meses;
    • o coeficiente isola o efeito esperado daquele mês.
    Exemplo de one-hot encoding 
    data        éFev  éMar  éAbr
2026-02-09     1     0     0
2026-03-16     0     1     0
2026-04-20     0     0     1
2026-05-11     0     0     0

    Se o coeficiente de éMar for positivo, março adiciona demanda esperada em relação ao mês de referência.

    Sazonalidade como OLS

    Modelo mental 
    emprestimos =
    tendencia
  + efeito_fevereiro * éFevereiro
  + efeito_marco * éMarço
  + efeito_abril * éAbril
  + erro

    Quando éMarço = 1: o efeito de março entra na previsão.

    Quando éMarço = 0: aquele efeito não entra.

    A ideia central é controlar a tendência e medir o desvio típico causado por cada período conhecido.

    Quando OLS é uma boa ideia?

    • Quando já sabemos quais períodos podem mudar a demanda.
    • Exemplo: semana de prova, fechamento de bimestre, férias ou recesso.
    • Cada evento vira uma coluna binária: éProva, éRecesso.
    • O coeficiente estima o efeito médio daquele evento.
    Exemplo 
    emprestimos =
    tendencia
  + 18 * éSemanaDeProva
  - 22 * éRecesso
  + erro

    Aqui a sazonalidade é guiada pelo calendário acadêmico, não apenas por um ciclo automático.

    OLS em Python: períodos conhecidos

    import statsmodels.api as sm df = pd.read_csv("dados/aula14/emprestimos.csv", parse_dates=["semana"]) df["t"] = range(len(df)) df["mes"] = df["semana"].dt.month_name() df["eh_prova"] = ( df["semana"].between("2026-03-09", "2026-03-23") | df["semana"].between("2026-06-09", "2026-06-23") | df["semana"].between("2026-09-09", "2026-09-23") | df["semana"].between("2026-11-09", "2026-11-23") ).astype(int)df["eh_recesso"] = df["semana"].between("2026-07-01", "2026-07-21").astype(int) dummies_mes = pd.get_dummies(df["mes"], prefix="mes", drop_first=True, dtype=int) X = pd.concat([df[["t", "eh_prova", "eh_recesso"]], dummies_mes], axis=1) X = sm.add_constant(X) modelo = sm.OLS(df["emprestimos"], X).fit() modelo.params

    Aqui os coeficientes estimam tendência, efeito de prova, efeito de recesso e diferença média entre meses.

    Modelo aditivo: observado = tendência + sazonalidade + resíduo.

    Leitura prática: primeiro entenda a direção geral, depois veja quais semanas ficam acima ou abaixo do esperado.

    Por que period=4?

    • Os dados estão em semanas.
    • Queremos procurar um padrão que se repete a cada 4 semanas.
    • Esse período pode representar um ciclo mensal aproximado.
    • Se o dado fosse mensal e o ciclo fosse anual, usaríamos period=12.
    Chamada principal 
    decomp = seasonal_decompose(
    serie,
    model="additive",
    period=4
)

    O período é uma hipótese sobre o ciclo do problema. Ele deve vir do contexto, não do chute automático.

    OLS ou decomposição clássica?

    Situação Boa escolha Motivo

    há um ciclo regular a cada 4 semanas seasonal_decompose(period=4) o padrão se repete de forma regular
    quero previsão operacional séria modelo temporal mais completo precisa validar erro fora da amostra

    Forecast simples: a conta

    Últimas quatro semanas 
    ultimas = serie.tail(4)

2026-05-11    68
2026-05-18    84
2026-05-25    91
2026-06-01    77

    O forecast usa a inclinação recente:

    passo = (77 - 68) / 3 = 3

    Próxima semana: 77 + 3 = 80

    Depois: 83, 86, 89

    É uma extrapolação linear curta: útil para explicar a lógica, mas fraca como modelo final.

    Código linha por linha

    Linha O que faz Por que importa
    seasonal_decompose(...) separa tendência, sazonalidade e resíduo transforma uma linha confusa em partes interpretáveis
    serie.tail(4) pega as quatro semanas mais recentes usa o comportamento mais próximo do presente
    passo calcula a inclinação média recente define quanto a projeção cresce por semana
    pd.date_range(...) cria as próximas datas semanais mantém o forecast com índice temporal
    pd.Series(...) junta valores previstos e datas gera uma saída legível como série temporal

    Decomposição e forecast em Python

    from statsmodels.tsa.seasonal import seasonal_decompose decomp = seasonal_decompose( serie, model="additive", period=4 ) ultimas = serie.tail(4) passo = (ultimas.iloc[-1] - ultimas.iloc[0]) / 3 forecast = [ serie.iloc[-1] + passo * i for i in range(1, 5) ] forecast_index = pd.date_range( serie.index[-1] + pd.Timedelta(days=7), periods=4, freq="7D" ) forecast_simples = pd.Series( forecast, index=forecast_index, name="forecast" ) print(decomp.trend.dropna().tail(6).round(1)) print(decomp.seasonal.head(4).round(1).to_numpy()) print(forecast_simples.round().astype(int))

    Interpretação: tendência explica a direção geral; sazonalidade mostra semanas acima/abaixo do esperado; forecast é hipótese de planejamento, não certeza.

    Análise de texto e outros dados que pedem cuidado especial

    Como lidar com dados que não são números ou categorias simples?

    • Texto precisa ser limpo, tokenizado e transformado em números.
    • A análise pode ser descritiva, classificação ou agrupamento de temas.
    • Leia amostras manualmente antes de confiar em métricas automáticas.

    Texto em Python: TF-IDF + clusters

    import pandas as pd import plotly.express as px from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.cluster import KMeans from sklearn.decomposition import TruncatedSVD df = pd.read_csv("dados/aula14/respostas.csv") textos = df["resposta_aberta"].fillna("") vetor = TfidfVectorizer( lowercase=True, stop_words=["de", "da", "do", "e", "a", "o", "para"] ) X = vetor.fit_transform(textos) clusters = KMeans( n_clusters=4, random_state=42, n_init="auto" ).fit_predict(X) coords = TruncatedSVD(n_components=2, random_state=42).fit_transform(X) df["cluster"] = clusters.astype(str) df["x"] = coords[:, 0] df["y"] = coords[:, 1] fig = px.scatter( df, x="x", y="y", color="cluster", hover_data=["resposta_aberta"] ) fig.show()

    A posição 2D ajuda a visualizar. Os nomes dos clusters vêm das palavras fortes e da leitura de exemplos reais.

    Chamados de suporte acadêmico: abertura, tipo_demanda, canal, turma, prioridade, tempo_resposta, resolvido.

    Pedido do professor

    "Quero identificar quais tipos de demanda demoram mais e quais têm maior chance de não serem resolvidas."

    Entregável esperado

    Modelo interpretável, variáveis mais importantes e propostas para priorizar atendimentos críticos.

    Cartão 3: manutenção

    Dataset

    Medições de máquinas: temperatura, vibração, corrente, data, operador, registro de falha.

    Pedido do professor

    "Quero saber quais leituras parecem fora do padrão e se isso antecipa falhas."

    Modelo de resposta do grupo

    Item Resposta curta
    Pergunta principal O que queremos descobrir, prever ou comparar?
    Variável alvo Existe? Qual é o tipo?
    Tipo de análise descritiva, comparação, regressão, classificação, clustering, PCA, anomalia, tempo ou texto
    Métodos candidatos 2 ou 3 métodos, não uma lista infinita
    Primeiro gráfico o gráfico que ajuda a entender antes de modelar
    Conclusão possível uma frase de decisão ou recomendação

    Exemplo de resposta: evasão

    Pergunta Quem tem maior risco de evasão?
    Alvo evadiu, categoria binária
    Análise classificação, com atenção a desbalanceamento
    Métodos regressão logística, árvore, Random Forest
    Primeiro gráfico taxa de evasão por faixa de frequência e por curso
    Conclusão possível estudantes com frequência abaixo de 60% e queda em notas parciais exigem intervenção prioritária

    Fase 6: checklist antes de apresentar

  • A pergunta principal está explícita?
  • O tipo de alvo foi identificado?
  • O primeiro gráfico realmente ajuda a entender o problema?
  • O método escolhido combina com a pergunta?
  • A métrica combina com o custo do erro?
  • A conclusão responde ao professor, não apenas ao algoritmo?

    O que evitar no projeto

    Evite

    "Usei três algoritmos e comparei acurácia."

    "O modelo deu 98%, então está ótimo."

    "Não sei o que significa o cluster 2."

    Prefira

    "A pergunta pede classificação binária."

    "Recall importa porque perder caso de risco é caro."

    "O cluster 2 representa uso intenso no período noturno."

    Resumo da Aula

  • Comece pela pergunta, não pelo algoritmo.
  • O tipo da variável alvo indica o caminho inicial.
  • Nem todo dataset pede machine learning supervisionado.
  • Séries temporais, texto e questionários exigem preparação específica.
  • Um bom projeto termina com uma conclusão útil para o problema real.
