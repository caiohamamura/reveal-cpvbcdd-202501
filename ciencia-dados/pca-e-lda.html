A Maldição da Dimensionalidade

Cenário: Você tem 100 features (dimensões) para prever uma variável-alvo.

Problema: quanto mais dimensões, mais dados precisamos!

📊 O que acontece?

  • Modelo pode overfitar
  • Distância entre pontos fica similar
  • Dados ficam esparsos
  • Custo computacional aumenta

💡 A solução

  • Reduzir dimensões
  • Preservar informação importante
  • Visualizar em 2D/3D
  • Melhorar performance do modelo

    PCA: Principal Component Analysis

    Definição: Técnica não-supervisionada que encontra novas direções (componentes) que maximizam a variância.

    🎯 Objetivo

    • Encontrar direções de máxima variância
    • Componentes são ortogonais (perpendiculares)
    • PC1 = maior variância, PC2 = segunda maior, etc.

    📈 Características

    • Não usa labels (não-supervisionado)
    • Reduz para qualquer k dimensões
    • Cada PC explica parte da variância total

    • Uma linha vertical (PC1) captura a altura — máxima variância
    • Uma linha horizontal (PC2) captura a largura — segunda maior
    • Juntas, representam o rosto com menos informação!

    LDA: Linear Discriminant Analysis

    Definição: Técnica supervisionada que encontra direções que maximizam a separação entre classes.

    🎯 Objetivo

    • Maximizar distância entre classes
    • Minimizar variância dentro das classes
    • Usado para classificação

    📈 Características

    • Usa labels (supervisionado)
    • Máximo de K-1 componentes (K = classes)
    • Intuitivamente: "poser pessoas para foto"

    PCA vs LDA: Comparação

    Característica PCA LDA
    Supervisionado? ❌ Não ✅ Sim
    Usa labels? Não Sim
    Objetivo Maximizar variância Maximizar separação
    Dimensões de saída Qualquer k Máximo K-1
    Uso principal Redução, visualização Classificação

    Edge Cases e Limitações

    Quando PCA e LDA falham

    Limitações do PCA

    • Dados não-lineares: PCA assume relações lineares entre variáveis
    • Sensibilidade a outliers: Um outlier pode distorcer os primeiros componentes
    • Variância ≠ Informação: Alta variância pode ser apenas ruído
    • Interpretabilidade: Autovetores são difíceis de interpretar com muitas features
    • Dados categóricos: PCA assume variáveis contínuas

    💡 Solução: Para dados não-lineares, use t-SNE, UMAP ou Kernel PCA

    Limitações do LDA

    • Máximo K-1 componentes: Limitado pelo número de classes
    • Suposição de normalidade: Assume distribuição normal multivariada
    • Classes desbalanceadas: Funciona mal quando uma classe tem 90% dos dados
    • Overfitting: Com muitas features e poucas amostras, overfita facilmente
    • Boundaries lineares: Só captura relações lineares

    ⚠️ Importante: Verifique normalidade multivariada antes de usar LDA

    Técnicas Não-Lineares

    Quando PCA e LDA não são suficientes

    Quando Usar Técnicas Não-Lineares?

    Dados não-lineares:

    • Manifolds curvos
    • Padrões circulares ou esféricos
    • Relações complexas entre variáveis

    Glossário e Erros Comuns

    Termos técnicos e como evitá-los

    Glossário Rápido

    Centralizar

    Subtrair a média (mover dados para origem)

    Padronizar

    Centralizar + dividir pelo desvio (média=0, dp=1)

    Autovetor

    Vetor que não muda de direção após transformação

    Autovalor

    Escala do autovetor após transformação

    Variância explicada

    Quanto cada componente explica da variância total

    Overfitting

    Modelo que memoriza ruído em vez de aprender padrão

    5 Erros Comuns (e como evitá-los)

    • 1. Esquecer de padronizar: Features com maior escala dominam → Sempre use StandardScaler()
    • 2. Usar labels no PCA: Vazamento de dados → PCA não deve receber y
    • 3. Escolher componentes demais: Overfitting → Use gráfico de cotovelo ou 80-95%
    • 4. Não separar treino/teste: Métricas otimistas → Aplique PCA só no treino
    • 5. Confundir centralizar com padronizar: Resultados incorretos

    💡 Lembre: Centralizar = média 0, dp original | Padronizar = média 0, dp 1

    Centralizar vs Padronizar

    CENTRALIZAR

    Move os dados para a origem

    PADRONIZAR

    Mesma escala para todas as features

    Seção 5: Conclusão e Recursos

    Resumo e próximos passos

    Resumo: O que Aprendemos

    • PCA = redução de dimensionalidade não-supervisionada
    • LDA = redução + classificação supervisionada
    • Ambos são lineares (assumem relações lineares)
    • PCA: maximize variância (não usa labels)
    • LDA: maximize separação entre classes (usa labels)

    Regra prática:
    Sem labels → PCA
    Com labels + classificação → LDA

    Quando Usar Cada Técnica?

    Cenário Técnica Recomendada
    Visualizar dados em 2D/3D PCA
    Classificação com labels LDA
    Pré-processamento para ML PCA ou LDA
    Não tem labels disponíveis PCA
    Muitas features (maldição dimensionalidade) PCA
