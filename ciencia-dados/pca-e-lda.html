A Maldição da Dimensionalidade

Cenário: Você tem 100 features (dimensões) para prever uma variável-alvo.

Problema: quanto mais dimensões, mais dados precisamos!

📊 O que acontece?

  • Modelo pode overfitar
  • Distância entre pontos fica similar
  • Dados ficam esparsos
  • Custo computacional aumenta

💡 A solução

  • Reduzir dimensões
  • Preservar informação importante
  • Visualizar em 2D/3D
  • Melhorar performance do modelo

    PCA: Principal Component Analysis

    Definição: Técnica não-supervisionada que encontra novas direções (componentes) que maximizam a variância.

    🎯 Objetivo

    • Encontrar direções de máxima variância
    • Componentes são ortogonais (perpendiculares)
    • PC1 = maior variância, PC2 = segunda maior, etc.

    📈 Características

    • Não usa labels (não-supervisionado)
    • Reduz para qualquer k dimensões
    • Cada PC explica parte da variância total

    Seção 2: A Matemática do PCA

    Covariância, autovetores e decomposição espectral

    • Matriz de Covariância: definição e cálculo
    • Autovetores e Autovalores: interpretação geométrica
    • Decomposição Espectral: Cov = QΛQᵀ

    ⏱️ Duração: ~45 minutos

    Matriz de Covariância — Definição Formal

    Definição: Cov(X) = E[(X - μ)(X - μ)ᵀ]

    Onde X é a matriz de dados, μ é o vetor de médias

    📐 O que mede?

    • Como cada feature varia consigo mesma (variância)
    • Como duas features variam juntas (covariância)
    • Matriz quadrada n×n (n = número de features)

    📊 Exemplo 2x2

    Cov = | Var(X₁)   Cov(X₁,X₂) |
      | Cov(X₂,X₁) Var(X₂)  |

    Matriz de Covariância — Cálculo Passo-a-Passo

    1. Centralizar dados: Xc = X - μ
    2. Calcular produto externo: Xcᵀ × Xc
    3. Normalizar por (n-1): Cov = XcᵀXc / (n-1)

    NumPy: np.cov(X.T) ou (Xc.T @ Xc) / (n-1)

    Autovetores e Autovalores — Definição

    Definição: Av = λv

    Onde A é a matriz de covariância, λ é autovalor, v é autovetor

    🔍 Interpretação

    • Autovetor (v): direção que não muda após transformação por A
    • Autovalor (λ): quanto a direção é "esticada"
    • Autovetores são ortogonais entre si

    Propriedade: Autovetores de Cov são as direções principais!

    Decomposição Espectral: Cov = QΛQᵀ

    Fórmula: Cov = Q Λ Qᵀ

    Q = Autovetores
    (matriz ortogonal)

    Λ = Autovalores
    (matriz diagonal)

    Qᵀ = Transposta
    (inversa de Q)

    Exemplo Numérico 2×2

    Resultado: PC1 (maior λ) explica a maior parte da variância!

    Por que Autovalores Maiores = Mais Importantes?

    Variância Explicada: λᵢ / Σλⱼ

    • λ maior = dados mais espalhados nessa direção = mais informação
    • λ menor = dados mais concentrados = menos informação (ruído?)
    • PC1 com λ₁=4.5 explica mais que PC2 com λ₂=0.8

    Seção 3: Quando a Matriz Não é Invertível

    Singularidade, pseudoinversa e regularização

    Quando uma Matriz é Invertível?

    Condição: det(Cov) ≠ 0 (determinante não-zero)

    • Matriz definida positiva: todos autovalores > 0
    • Não-singular: existe matriz inversa Cov⁻¹
    • Problemas: features redundantes, mais features que amostras

    ⚠️ Caveat: Matrizes quase-singulares são numericamente instáveis!

    Pseudoinversa e Regularização

    📐 Pseudoinversa (Moore-Penrose)

    Funciona mesmo quando det=0

    🛡️ Regularização (Tikhonov)

    Stabiliza computação, levemente alterando autovetores

    Edge Cases e Limitações

    Quando o PCA falha

    Limitações do PCA

    • Dados não-lineares: PCA assume relações lineares entre variáveis
    • Sensibilidade a outliers: Um outlier pode distorcer os primeiros componentes
    • Variância ≠ Informação: Alta variância pode ser apenas ruído
    • Interpretabilidade: Autovetores são difíceis de interpretar com muitas features
    • Dados categóricos: PCA assume variáveis contínuas

    💡 Solução: Para dados não-lineares, use t-SNE, UMAP ou Kernel PCA

    Técnicas Não-Lineares

    Quando Usar Técnicas Não-Lineares?

    Glossário e Erros Comuns

    Termos técnicos e como evitá-los

    Glossário Rápido

    Centralizar

    Subtrair a média (mover dados para origem)

    Padronizar

    Centralizar + dividir pelo desvio (média=0, dp=1)

    Autovetor

    Vetor que não muda de direção após transformação

    Autovalor

    Escala do autovetor após transformação

    Variância explicada

    Quanto cada componente explica da variância total

    Overfitting

    Modelo que memoriza ruído em vez de aprender padrão

    5 Erros Comuns (e como evitá-los)

    • 1. Esquecer de padronizar: Features com maior escala dominam → Sempre use StandardScaler()
    • 2. Usar labels no PCA: Vazamento de dados → PCA não deve receber y
    • 3. Escolher componentes demais: Overfitting → Use gráfico de cotovelo ou 80-95%
    • 4. Não separar treino/teste: Métricas otimistas → Aplique PCA só no treino
    • 5. Confundir centralizar com padronizar: Resultados incorretos

    💡 Lembre: Centralizar = média 0, dp original | Padronizar = média 0, dp 1

    Centralizar vs Padronizar

    CENTRALIZAR

    Move os dados para a origem

    PADRONIZAR

    Mesma escala para todas as features
