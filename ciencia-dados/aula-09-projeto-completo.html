Validação Cruzada (k-fold) 🔄

  • k=5 ou k=10 folds
  • Cada fold: parcela diferente como validação
  • Resultado: média das k folds = score estável
  • Mais robusto que single split

Por que k-fold?

Single split varia muito. k-fold reduz variância na estimativa.

Modelo RandomForest 🌲🌲🌲

  • Ensemble de múltiplas árvores de decisão
  • Bagging: cada árvore treina em amostra bootstrap
  • Votação majoritária: classe mais votada vence
  • Menos overfitting que árvore única

Métricas de Avaliação 📊

MétricaFórmulaInterpretação
AccuracyTP+TN / Total% total de acertos
PrecisionTP / (TP+FP)Das classificadas risco, quantas eram?
RecallTP / (TP+FN)Das que eram risco, quantas encontrou?
F1 Score2×P×R / (P+R)Harmônico entre precision e recall

⚠️ Sempre no teste — nunca no treino!

Feature Importance 📊

  • Mede quanto cada feature reduz impureza de Gini
  • Features no topo = mais decisivas
  • Ajuda a interpretar e simplificar o modelo

Exemplo German Credit:

  • Duration — duração do crédito
  • Amount — valor do crédito
  • Age — idade

Para que serve?

  • Identificar features mais importantes
  • Remover features irrelevantes
  • Explicar modelo para stakeholders

Ética: O Caso COMPAS ⚖️

O que é COMPAS?

Correctional Offender Management Profiling for Alternative Sanctions — algoritmo usado na justiça criminal dos EUA para prever reincidência.

Descoberta da ProPublica (2016):

  • Negros: 45% chance de alto risco mesmo sem reincidir
  • Brancos: 23% chance

Mesmo controlando para histórico criminal!

Questões para Discussão:

  • Se o modelo erra mais para um grupo, isso é justo?
  • Quem é responsável quando o algoritmo erra?
  • Podemos usar modelos assim em decisões de justiça?
  • Como mitigar vieses algorítmicos?
    •

    Fairness Metrics: Além de accuracy, usar Precision, Recall e Disparate Impact (razão entre taxas de grupos). Se taxa_negativos/taxa_positivos < 0.8, há possível viés.
