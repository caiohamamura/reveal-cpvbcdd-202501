    • 80/20 ou 70/30 — trade-off: mais treino vs mais validação
    • random_state=42 — reprodutibilidade (mesmo split toda vez)
    • stratify=y — manter proporção da classe (importante para dados desbalanceados)

    Por que stratify=y?

    Dataset desbalanceado: 70% bom, 30% ruim

    Sem stratify:

    • Teste pode ter 90%/10% por azar
    • Modelo parece melhor do que é

    Com stratify:

    • Teste mantém 70%/30%
    • Avaliação justa

    Validação Cruzada (k-fold) 🔄

    • k=5 ou k=10 (mais robusto que single split)
    • Cada fold: parcela diferente como validação
    • Resultado: média das k folds = score mais estável

    Single split muda muito → k-fold estabiliza

    Visualizando o Overfitting

    Quanto mais profundo → treino sobe, teste estagna ou cai. O ponto ideal está no meio.

    O Trade-off Viés-Variância

    Underfitting

    • Alto viés (bias)
    • Modelo simples demais
    • Não captura padrões
    • Ex: max_depth=1

    Equilíbrio Ideal

    • Viés e variância equilibrados
    • Generaliza bem
    • Captura padrões reais
    • Ex: max_depth=4

    Overfitting

    • Alta variância
    • Modelo complexo demais
    • Decora o treino
    • Ex: max_depth=None

    Ética em Data Science ⚖️

    • "Se o modelo erra mais em pessoas de baixa renda, isso é justo?"
    • "Quando usamos dados históricos, estamos perpetuando vieses?"
    • "O teste invisível existe para proteger o modelo de quê?"
    • "Quem é responsável quando o modelo erra?"

    Viés de Seleção e Justiça

    • Viés de seleção no split — dados não representativos
    • Modelo bom não é só acurácia alta
    • É acurácia alta para TODOS os grupos
