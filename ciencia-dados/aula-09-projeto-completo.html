⚠️ Por que usar strata?

  • Se classes são desbalanceadas (70% bom, 30% mau), split aleatório pode gerar viés
  • Sem strata: treino pode ter 80% bom, teste 50% bom = modelo com viés
  • Com strata: proporções mantidas em ambos os conjuntos

Validação Cruzada (k-fold) 🔄

  • k=5 ou k=10 folds
  • Cada fold: parcela diferente como validação
  • Resultado: média das k folds = score estável
  • Mais robusto que single split

Por que k-fold?

Single split varia muito. k-fold reduz variância na estimativa.

Modelo RandomForest 🌲🌲🌲

  • Ensemble de múltiplas árvores de decisão
  • Bagging: cada árvore treina em amostra bootstrap
  • Votação majoritária: classe mais votada vence
  • Menos overfitting que árvore única

Contexto Histórico

  • Paper de Leo Breiman (2001): "Random Forests"
  • Baseado em dois pilares: bagging + random feature selection
  • Um dos primeiros algoritmos de ensemble

Por que é o padrão de comparação?

  • Robusto: funciona bem "out of the box"
  • Não precisa de muito tuning
  • Combina poder preditivo com interpretabilidade
  • Frequentemente usado como baseline em competitions (Kaggle, etc.)
  • Novas técnicas precisam "bater" o RF para provar valor

⚠️ Limitações Importantes

Não extrapola! — Só prevê dentro do range dos dados de treino.
Se treino tem Age 20-60, não consegue prever Age=80.
  • Interpretabilidade: individuais vs globais
  • Pode ter overfit com muitas árvores (raro, mas possível)

AUC-ROC Explained 📈

O que é AUC?

Área sob a curva ROC — mede a capacidade discriminativa do modelo.

AUCInterpretação
= 0.5Azar (classificador aleatório)
< 0.5Invertido (preve ao contrário!)
> 0.5Bom — mais perto de 1, melhor
> 0.9Excelente (cuidado com overfitting)

Pro tip: AUC = 0.5 significa que o modelo não sabe distinguir as classes. AUC > 0.8 geralmente é considerado bom.

Feature Importance 📊

  • Mede quanto cada feature reduz impureza de Gini
  • Features no topo = mais decisivas
  • Ajuda a interpretar e simplificar o modelo

Exemplo German Credit:

  • Duration — duração do crédito
  • Amount — valor do crédito
  • Age — idade

Para que serve?

  • Identificar features mais importantes
  • Remover features irrelevantes
  • Explicar modelo para stakeholders

Feature Selection 🎯

Nem toda feature ajuda — escolher bem melhora modelo e interpretabilidade.

Métodos Estatísticos:

  • AIC/BIC: penaliza complexidade (quanto menor, melhor)
  • p-value: testa significância de cada feature
  • VIF: detecta multicolinearidade

Conhecimento de Domínio:

  • Expertise do domínio é insubstituível
  • Remove features que não fazem sentido
  • Adiciona features com lógica de negócio

⚠️ Armadilhas:

  • Não adicionar features sem embasamento
  • Não remover só porque "parece irrelevant"
  • Validação depois de mudar o conjunto

🌲 Robustez do RandomForest:

  • Árvores ignoram ruído: features irrelevantes são automaticamente "esquecidas" nas divisões
  • Não precisa de normalização: RF baseado em splits categóricos, não em distâncias
  • Feature selection pode ser mais arte que ciência: muitas combinações plausíveis podem performar similarmente — mas só testando você sabe qual funciona. Por isso: experimente, não assuma!

📊 Paramétrico vs Não-Paramétrico:

  • Regressão Linear: paramétrica — assume distribuição normal dos resíduos
  • RandomForest: não-paramétrico — não assume distribuição, funciona com qualquer formato
  • Vantagem RF: não precisa de normalização, outliers têm menos impacto

O que é COMPAS?
