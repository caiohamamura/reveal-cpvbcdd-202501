    Por que stratify?

    Dataset desbalanceado: 70% bom, 30% ruim

    Sem estratificação:

    • Teste pode ter 90%/10% por azar
    • Modelo parece melhor do que é

    Com estratificação:

    • Teste mantém 70%/30%
    • Avaliação justa

    Validação Cruzada (k-fold) 🔄

    • k=5 ou k=10 (mais robusto que single split)
    • Cada fold: parcela diferente como validação
    • Resultado: média das k folds = score mais estável

    Single split muda muito → k-fold estabiliza

    Overfitting vs Underfitting

    Underfitting

    • Alto viés (bias)
    • Modelo simples demais
    • Não captura padrões
    • Ex: max_depth=1

    Equilíbrio Ideal

    • Viés e variância equilibrados
    • Generaliza bem
    • Captura padrões reais
    • Ex: max_depth=4

    Overfitting

    • Alta variância
    • Modelo complexo demais
    • Decora o treino
    • Ex: max_depth=NULL

    Visualizando o Overfitting

    Onde os Alunos Travam 💥

    • Fazem split ANTES do tratamento → vazamento de informação
    • Avaliam no dado de treino achando que é o teste real
    • Esquecem estratificação em dados desbalanceados

    Ordem correta:

    Tratamento → Split → Validação Cruzada → Teste Final

    O Problema de Uma Árvore

    • Uma árvore de decisão sofre de alta variância
    • Mudar 1 registro no treino → árvore completamente diferente
    • Tende ao overfitting se ficar muito profunda
    • Sensível a outliers e ruído

    Solução: construir muitas árvores e combinar as predições.

    Ensemble e Bagging 🎒

    Ensemble = combinar vários modelos

    • Modelos fracos individuais
    • Combinação forte coletiva
    • Reduz variância sem aumentar viés

    Bagging (Bootstrap Aggregating)

    • Amostras com reposição (bootstrap)
    • Cada árvore vê um subset diferente
    • Predição por votação majoritária

    • Feature randomness — cada split considera só √p features
    • Votação — classe mais votada ganha

    Por que Random Forest é melhor?

    Árvore Única

    • Alta variância
    • Sensível ao treino
    • Fácil de interpretar
    • Overfitting fácil

    Random Forest

    • Baixa variância (média de N árvores)
    • Robusta a ruído e outliers
    • Menos overfitting
    • Feature importance natural

    Feature Importance 📊

    • Mede quanto cada feature reduz a impureza nas árvores
    • Features no topo = mais decisivas para a predição
    • Útil para interpretar o modelo e simplificar features

    Árvore vs Random Forest

    Random Forest geralmente tem acurácia maior e menor overfitting.

    Ética em Data Science ⚖️

    • "Se o modelo erra mais em pessoas de baixa renda, isso é justo?"
    • "Quando usamos dados históricos, estamos perpetuando vieses?"
    • "Random Forest é menos transparente — isso é um problema?"
    • "Quem é responsável quando o modelo erra?"

    Viés de Seleção e Justiça

    • Viés de seleção no split — dados não representativos
    • Modelo bom não é só acurácia alta
    • É acurácia alta para TODOS os grupos

    Resumo da Aula

  • Pipeline completo: Dados → Limpeza → Features → Split → Treino → Validação → Teste
  • Split com estratificação mantém proporção das classes
  • Validação cruzada é mais robusta que single split
  • Teste final é blindado — olhar só uma vez
  • Overfitting: modelo decora treino, falha no teste
  • Random Forest: ensemble de árvores com bagging e feature randomness
  • Feature importance ajuda a interpretar o modelo
  • Ética: o modelo afeta vidas reais
