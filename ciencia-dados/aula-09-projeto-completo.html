Validação Cruzada (k-fold) 🔄

  • k=5 ou k=10 folds
  • Cada fold: parcela diferente como validação
  • Resultado: média das k folds = score estável
  • Mais robusto que single split

Por que k-fold?

Single split varia muito. k-fold reduz variância na estimativa.

Contexto Histórico

  • Paper de Leo Breiman (2001): "Random Forests"
  • Baseado em dois pilares: bagging + random feature selection
  • Um dos primeiros algoritmos de ensemble

Por que é o padrão de comparação?

  • Robusto: funciona bem "out of the box"
  • Não precisa de muito tuning
  • Combina poder preditivo com interpretabilidade
  • Frequentemente usado como baseline em competitions (Kaggle, etc.)
  • Novas técnicas precisam "bater" o RF para provar valor

⚠️ Limitações Importantes

Não extrapola! — Só prevê dentro do range dos dados de treino.
Se treino tem Age 20-60, não consegue prever Age=80.
  • Interpretabilidade: individuais vs globais
  • Pode ter overfit com muitas árvores (raro, mas possível)

Métricas de Avaliação 📊

Métrica Fórmula Interpretação
Accuracy TP+TN / Total % total de acertos
Precision TP / (TP+FP) Das classificadas risco, quantas eram?
Recall TP / (TP+FN) Das que eram risco, quantas encontrou?
F1 Score 2×P×R / (P+R) Harmônico entre precision e recall
AUC-ROC Área sob curva ROC Probabilidade de classificar positivo maior que negativo

💡 as.factor(): yarstick exige que truth seja fator — classe positiva deve ser o segundo nível para event_level = "second"

AUC-ROC Explained 📈

O que é AUC?

Área sob a curva ROC — mede a capacidade discriminativa do modelo.

AUC Interpretação
= 0.5 Azar (classificador aleatório)
< 0.5 Invertido (preve ao contrário!)
> 0.5 Bom — mais perto de 1, melhor
> 0.9 Excelente (cuidado com overfitting)

Pro tip: AUC = 0.5 significa que o modelo não sabe distinguir as classes. AUC > 0.8 geralmente é considerado bom.

Feature Importance 📊

  • Mede quanto cada feature reduz impureza de Gini
  • Features no topo = mais decisivas
  • Ajuda a interpretar e simplificar o modelo

Feature Selection 🎯

Escolher features certas melhora modelo, interpretabilidade e reduz overfitting.

Métodos Automáticos:

  • Stepwise (AIC/BIC): adiciona/remove features iterativamente
  • Random Forest Importance: rankeia por redução de impureza
  • LASSO (L1): zera coeficientes menos importantes

Feature Selection: Conhecimento de Domínio 🧠

Expertise humana é insubstituível — máquinas não entendem contexto de negócio.

  • Remove features sem sentido lógico
  • Adiciona features com lógica de negócio
  • Valida se rankings automáticos fazem sentido
  • Identifica confounders e causalidade

💡 Combinação ideal:

Use métodos automáticos para screening e conhecimento de domínio para validação final.

Stepwise AIC/BIC: Como Funciona

# 1. Começa com modelo vazio modelo_null <- glm(Target ~ 1, data=treino, family="binomial" ) # 2. Define modelo completo modelo_full <- glm(Target ~ ., data=treino, family="binomial" ) # 3. Stepwise iterativo modelo_step <- step(modelo_null, scope=list(lower=modelo_null, upper=modelo_full), direction="both" , k=log(nrow(treino))) # BIC # 4. Avalia BIC(modelo_step)

  2. Passo 2: Define modelo com todas features
  3. Passo 3: Adiciona/remove iterativamente
  4. Passo 4: Extrai melhor BIC

⚠️ AIC/BIC: Quando Usar?

AIC e BIC só funcionam com modelos que têm likelihood

✅ Funcionam:

  • Regressão logística (glm)
  • Regressão linear
  • Modelos com distribuição definida

❌ Não funcionam:

  • RandomForest
  • KNN
  • SVM

→ Usar AUC ou cross-validation score

Métodos de Seleção de Features 🔬

Não existe método perfeito — cada um tem prós e contras.

Método Como funciona ✅ Vantagem ⚠️ Desvantagem
Stepwise (AIC/BIC) Adiciona/remove features iterativamente Simples, interpretável Pode parar em local optimum
Random Forest Baseado em redução de impureza de Gini Captura relações não-lineares Ignora correlação entre features
LASSO (L1) Zera coeficientes menos importantes Seleciona + regulariza Requer muitos zeros (sparsity)
RFE Elimina features uma por vez Funciona com qualquer modelo Muito lento (O(n²))

💡 Dica: Para começar, use Random Forest Importance ou Correlation filter — são rápidos e dão boa baseline.

🌲 Robustez do RandomForest

  • Ignora ruído: features irrelevantes são "esquecidas" nas divisões
  • Não precisa de normalização: splits categóricos, não distâncias
  • Paper de Breiman (2001): "Random Forests" — ensemble robusto

⚠️ Não extrapola! Só prevê dentro do range dos dados de treino.

  • Feature selection: mais arte que ciência — teste diferentes combinações

Ética: O Caso COMPAS ⚖️

O que é COMPAS?

Correctional Offender Management Profiling for Alternative Sanctions — algoritmo usado na justiça criminal dos EUA para prever reincidência.

Descoberta da ProPublica (2016):

  • Negros: 45% chance de alto risco mesmo sem reincidir
  • Brancos: 23% chance

Mesmo controlando para histórico criminal!
