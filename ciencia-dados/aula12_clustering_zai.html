    Clustering Hierárquico

    Cada ponto começa sozinho → funde os mais próximos → constrói uma árvore

    Abordagem Aglomerativa

  • Cada ponto é seu próprio cluster
  • Calcule a distância entre todos os pares de clusters
  • Funda os dois clusters mais próximos
  • Repita até ter um único cluster
    •

    Métodos de Linkage

    MétodoComo mede distância entre clusters
    SingleMenor distância entre pontos dos dois clusters
    CompleteMaior distância entre pontos dos dois clusters
    AverageMédia das distâncias entre todos os pares
    WardMinimiza a variância total dentro dos clusters

    Dendrograma

    Árvore que mostra todas as fusões. A altura indica a distância em que os clusters se uniram.

    Como Ler o Dendrograma

    • Eixo Y: distância de fusão
    • Fusões baixas: clusters similares
    • Fusões altas: clusters diferentes
    • Saltos grandes: separações naturais
    • Linha de corte: define o número de clusters

    Cortamos onde há um salto grande → K=4 clusters

    Resultado: 4 Clusters Hierárquicos (Ward)

    Hierárquico — Prós e Contras

    Prós

    • Dendrograma é intuitivo e visual
    • Não precisa definir K antes
    • Mostra estrutura completa dos dados

    Contras

    • Custo computacional O(n²) a O(n³)
    • Sensível ao método de linkage
    • Não pode desfazer fusões

    K-Means: O Algoritmo Mais Popular

    Divide os dados em K clusters usando centroides como representação

    O Algoritmo K-Means

  • Escolha K centroides iniciais (aleatórios)
  • Atribuição: cada ponto vai para o centroide mais próximo
  • Atualização: recalcular centroides como média dos pontos
  • Repetir até convergir (centroides não mudam)
    •

    K-Means Passo a Passo

    Dados originais — sem agrupamento

    Resultado: K-Means (K=4)

    Escolhendo o K: Cotovelo e Silhouette

    K-Means — Prós e Contras

    Prós

    • Rápido e escalável O(n·K·i)
    • Intuitivo — centroides como moldes
    • Poucos parâmetros (apenas K)

    Contras

    • Precisa definir K antes
    • Assume clusters esféricos
    • Sensível à inicialização
    • Não detecta ruído

    Quando K-Means Falha

    K-Means assume clusters esféricos. E quando não são?

    K-Means em Dados Não-Esféricos

    Precisamos de um método que entenda densidade, não distância ao centroide.

    DBSCAN: Agrupamento por Densidade

    "Esta região é densa o suficiente para ser um cluster?"

    Conceitos do DBSCAN

    • ε (epsilon): raio de vizinhança
    • min_samples: mínimo de pontos na vizinhança
    • Core point: tem ≥ min_samples vizinhos
    • Border point: vizinho de um core, mas não é core
    • Noise point: nem core nem border → ruído

    DBSCAN nas Luas — Formas Arbitrárias

    DBSCAN — Prós e Contras

    Prós

    • Não precisa definir K
    • Detecta formas arbitrárias
    • Identifica outliers automaticamente

    Contras

    • Sensível a ε e min_samples
    • Problemas com densidades variáveis
    • Alta dimensionalidade: distância perde sentido

    K-Means pergunta: "Qual centroide é o mais próximo?"

    DBSCAN pergunta: "Esta região é densa o suficiente?"
