    🎛️ Inline Keyboards vs Comandos /

    Comandos /

  • Usuário digita /ledon
  • Requer teclado físico
  • Menos intuitivo
  • Funciona em qualquer bot

    Inline Keyboards

  • Usuário clica em botões
  • Mais visual e fácil
  • Feedback imediato
  • Melhor experiência mobile

    🧰 Componentes Necessários

    📡 NodeMCU ESP8266

    • Placa principal com Wi-Fi
    • Wi-Fi: 802.11 b/g/n
    • GPIO para conexões
    • USB para programação

    🌡️ Sensor DHT22

    • Mede temperatura e umidade
    • Precisão: ±0.5°C
    • Ligação: 1-wire
    • Pino: GPIO4 (D2)

    💡 LED (on-board)

    • LED embutido no GPIO2
    • Já existe na NodeMCU!
    • Para teste inicial

    🔌 Fiação

    • Jumpers para conexões
    • Protoboard (opcional)
    • Cabo USB

    📍 Pinagem NodeMCU v2

    LED_BUILTIN = GPIO2 | DHT22 = GPIO4 (D2)

    🎮 Callbacks dos Botões Inline

    
// ========== CALLBACKS DOS BOTÕES ==========
void onLedOn(const TBMessage &queryMsg) {
  digitalWrite(LED_PIN, LOW);
  ledState = true;
  Serial.println("LED ON via callback");
  bot.endQuery(queryMsg, "💡 LED ligado!", true);
  String reply = "✅ *LED LIGADO*\n\n💡 Use os botões para controlar";
  bot.editMessage(queryMsg, reply, myInlineKbd);
}

void onLedOff(const TBMessage &queryMsg) {
  digitalWrite(LED_PIN, HIGH);
  ledState = false;
  Serial.println("LED OFF via callback");
  bot.endQuery(queryMsg, "⚫ LED desligado!", true);
  String reply = "⚫ *LED DESLIGADO*\n\n💡 Use os botões para controlar";
  bot.editMessage(queryMsg, reply, myInlineKbd);
}

void onStatus(const TBMessage &queryMsg) {
  Serial.println("Status via callback");
  sendStatus(queryMsg);
  bot.endQuery(queryMsg, "📊 Status atualizado!", true);
}
  • bot.endQuery(): Confirma o callback e mostra notificação
  • bot.editMessage(): Atualiza a mensagem com teclado

    📡 WiFi e Configuração TLS

    
// Conectar WiFi
WiFi.mode(WIFI_STA);
WiFi.begin(WIFI_SSID, WIFI_PASSWORD);

// Espera conexão
while (WiFi.status() != WL_CONNECTED) {
  delay(500);
  Serial.print(".");
}

#ifdef ESP8266
  // Sincronizar hora (necessário para certificado TLS)
  configTime(MYTZ, "time.google.com", 
             "time.windows.com", "pool.ntp.org");
  client.setSession(&session);
  client.setTrustAnchors(&certificate);
  client.setBufferSizes(1024, 1024);
#endif

// Inicializar bot
bot.setUpdateTime(1000);
bot.setTelegramToken(TELEGRAM_BOT_TOKEN);

if (bot.begin()) {
  Serial.println("Bot Telegram OK!");
} else {
  Serial.println("Falha no bot!");
}

    🔄 Loop Principal

    
void loop() {
  // LED pisca para indicar que está rodando
  static uint32_t ledTime = 0;
  if (millis() - ledTime > 1000) {
    ledTime = millis();
    digitalWrite(LED_PIN, !digitalRead(LED_PIN));
  }

  TBMessage msg;
  if (bot.getNewMessage(msg)) {
    MessageType msgType = msg.messageType;
    String msgText = msg.text;

    switch (msgType) {
      case MessageText:
        if (msgText.equalsIgnoreCase("/start")) {
          String reply = "🤖 *Bot IoT - ESP8266*\n\n";
          reply += "Use os botões abaixo para controlar o LED:";
          bot.sendMessage(msg, reply.c_str(), myInlineKbd);
        }
        else if (msgText.equalsIgnoreCase("/status")) {
          sendStatus(msg);
        }
        break;

      case MessageQuery:
        // Callbacks tratados automaticamente
        break;
    }
  }
}

    📊 Função: sendStatus()

    
void sendStatus(const TBMessage &msg) {
  String reply = "📊 *Status do Sistema*\n\n";
  
  float temp = dht.readTemperature();
  float humidity = dht.readHumidity();
  
  if (isnan(temp) || isnan(humidity)) {
    reply += "🌡️ Temperatura: Erro\n";
    reply += "💧 Umidade: Erro\n";
  } else {
    reply += "🌡️ Temperatura: " + String(temp, 1) + " °C\n";
    reply += "💧 Umidade: " + String(humidity, 1) + " %\n";
  }
  
  reply += "\n💡 LED: ";
  reply += ledState ? "*LIGADO*" : "*DESLIGADO*";
  reply += "\n📶 WiFi: " + String(WiFi.RSSI()) + " dBm";

  bot.sendMessage(msg, reply.c_str(), myInlineKbd);
}

    📟 Comandos e Botões

    Tipo Ação Descrição
    /start Boas-vindas Mostra teclado inline
    /status Status 🌡️ Temp + 💧 Umidade
    💡 LED ON Liga LED ✅ Callback automático
    ⚫ LED OFF Desliga LED ✅ Callback automático
    📊 Status Status ✅ Callback automático

    🔧 Atividade Prática

  • 1. Crie um bot no Telegram: busque @BotFather
  • 2. Use /newbot e siga as instruções
  • 3. Copie o TOKEN fornecido
  • 4. Cole no arquivo src/secrets.h
  • 5. Copie o certificado TLS para include/tg_certificate.h
  • 6. Compile: pio run
  • 7. Grave: pio run --target upload
  • 8. Envie /start para seu bot!

    📝 Resumo: AsyncTelegram2 vs UniversalTelegramBot

    Feature AsyncTelegram2 ✅ UniversalTelegramBot
    Bloqueante Não (async) Sim (blocking)
    Inline Keyboards Sim + callbacks Sim (basic)
    TLS Certificate Obrigatório Opcional (setInsecure)
    NTP Sync Necessário Não
    Velocidade Mais rápido Mais lento
